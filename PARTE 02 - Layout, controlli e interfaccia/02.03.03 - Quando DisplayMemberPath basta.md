# 02.03.03 - Quando `DisplayMemberPath` basta

`DisplayMemberPath` è una soluzione semplice, pulita e spesso sottovalutata. In molti casi è esattamente quello che serve, e usare qualcosa di più complesso sarebbe solo rumore.

Il punto importante è capire **quando basta davvero** e quando invece si sta forzando un controllo semplice a fare troppo.

## Che cosa fa `DisplayMemberPath`

`DisplayMemberPath` indica quale proprietà dell’oggetto deve essere visualizzata nella lista.

Esempio:

```xml
<ComboBox ItemsSource="{Binding Aree}"
          SelectedItem="{Binding AreaSelezionata}"
          DisplayMemberPath="Descrizione" />
```

Se ogni elemento della collezione è un oggetto `Area`, la `ComboBox` mostrerà il valore della proprietà `Descrizione`.

## Quando è la scelta migliore

`DisplayMemberPath` è ottimo quando:

- devi mostrare una sola proprietà;
- la proprietà è già pronta per essere letta;
- non hai bisogno di immagini, icone o più righe;
- non vuoi aggiungere complessità inutile.

## Caso tipico: liste anagrafiche semplici

Pensa a queste situazioni:

- elenco degli stati;
- elenco delle aree;
- elenco dei reparti;
- elenco delle priorità;
- elenco delle valute.

In tutti questi casi, mostrare una sola proprietà come `Descrizione` è spesso la soluzione più giusta.

## Esempio completo

### C#

```csharp
public class StatoDocumento
{
    public int Id { get; set; }
    public string Descrizione { get; set; } = "";
}

public List<StatoDocumento> Stati { get; } = new()
{
    new StatoDocumento { Id = 1, Descrizione = "Bozza" },
    new StatoDocumento { Id = 2, Descrizione = "Confermato" },
    new StatoDocumento { Id = 3, Descrizione = "Annullato" }
};

public StatoDocumento? StatoSelezionato { get; set; }
```

### XAML

```xml
<ComboBox Width="220"
          ItemsSource="{Binding Stati}"
          SelectedItem="{Binding StatoSelezionato}"
          DisplayMemberPath="Descrizione" />
```

## Perché qui è la scelta giusta

In questo caso:

- la lista è semplice;
- ogni elemento ha un solo testo da mostrare;
- l’utente non ha bisogno di vedere altro.

Usare un `ItemTemplate` qui sarebbe possibile, ma meno elegante. Aumenterebbe il codice senza dare un vero vantaggio.

## Vantaggi pratici di `DisplayMemberPath`

### Meno XAML

Il codice è breve, leggibile e facile da mantenere.

### Più chiarezza

Chi legge la vista capisce subito che la `ComboBox` mostra la proprietà `Descrizione`.

### Meno rischio di errore

Meno struttura visiva significa meno possibilità di rompere il layout o confondere il comportamento del controllo.

## Quando inizia a non bastare più

`DisplayMemberPath` smette di essere sufficiente quando devi mostrare:

- più proprietà insieme;
- più righe per ogni elemento;
- immagini;
- icone;
- formattazioni particolari;
- etichette secondarie.

Per esempio, se vuoi mostrare:

- nome cliente;
- città;
- codice;
- piccola immagine;

allora `DisplayMemberPath` non può bastare.

## Un confronto diretto

### Soluzione semplice con `DisplayMemberPath`

```xml
<ComboBox ItemsSource="{Binding Clienti}"
          DisplayMemberPath="RagioneSociale" />
```

### Soluzione che richiede `ItemTemplate`

```xml
<ComboBox ItemsSource="{Binding Clienti}">
    <ComboBox.ItemTemplate>
        <DataTemplate>
            <StackPanel>
                <TextBlock Text="{Binding RagioneSociale}" />
                <TextBlock Text="{Binding Citta}" FontSize="11" />
            </StackPanel>
        </DataTemplate>
    </ComboBox.ItemTemplate>
</ComboBox>
```

La domanda giusta non è “quale dei due è più potente?”, ma:

> “Mi serve davvero quella complessità in più?”

## Una regola utile

Se stai mostrando un solo testo leggibile, prova sempre prima con `DisplayMemberPath`.

Passa a `ItemTemplate` solo quando il controllo deve davvero rappresentare informazioni più ricche.

## Errore tipico

Un errore frequente è saltare subito al template “perché sembra più professionale”. In realtà, se il caso è semplice, il template rende il codice più lungo senza migliorare davvero la comprensione dell’utente.

In UI, più complesso non significa automaticamente migliore.

## Il punto da portare a casa

`DisplayMemberPath` basta in tutti i casi in cui la `ComboBox` deve visualizzare **una sola proprietà chiara e leggibile**. È la soluzione più semplice, più lineare e spesso anche la più corretta. In WPF conviene quasi sempre partire da qui e complicare solo quando serve davvero.

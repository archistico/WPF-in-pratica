# 02.04.06 - Collegare `MenuItem` e `Button` allo stesso comando

Qui si vede il vantaggio concreto del sistema dei comandi: più controlli diversi possono esporre **la stessa azione**.

## Esempio completo con `ApplicationCommands.Save`

### XAML

```xml
<Window x:Class="DemoMenuToolbar.MainWindow"
        xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation"
        xmlns:x="http://schemas.microsoft.com/winfx/2006/xaml"
        Title="Demo comandi" Height="250" Width="500">

    <Window.CommandBindings>
        <CommandBinding Command="ApplicationCommands.Save"
                        Executed="Save_Executed"
                        CanExecute="Save_CanExecute" />
    </Window.CommandBindings>

    <DockPanel>
        <Menu DockPanel.Dock="Top">
            <MenuItem Header="_File">
                <MenuItem Header="_Salva"
                          Command="ApplicationCommands.Save" />
            </MenuItem>
        </Menu>

        <ToolBarTray DockPanel.Dock="Top">
            <ToolBar>
                <Button Content="Salva"
                        Command="ApplicationCommands.Save" />
            </ToolBar>
        </ToolBarTray>

        <Grid>
            <TextBox x:Name="txtContenuto"
                     Margin="12"
                     AcceptsReturn="True"
                     TextWrapping="Wrap" />
        </Grid>
    </DockPanel>
</Window>
```

### C#

```csharp
private bool _datiModificati = true;

private void Save_Executed(object sender, ExecutedRoutedEventArgs e)
{
    MessageBox.Show("Salvataggio eseguito.");
    _datiModificati = false;
    CommandManager.InvalidateRequerySuggested();
}

private void Save_CanExecute(object sender, CanExecuteRoutedEventArgs e)
{
    e.CanExecute = _datiModificati;
}
```

## Che cosa sta succedendo

- `ApplicationCommands.Save` rappresenta l'azione logica di salvataggio;
- `CommandBinding` collega quel comando alla logica reale;
- sia il `MenuItem` sia il `Button` usano lo stesso comando;
- `CanExecute` decide se l'azione è disponibile.

## Perché questa soluzione è migliore

Rispetto a due `Click` distinti, qui ottieni:

- una sola logica di esecuzione;
- uno stato abilitato/disabilitato coerente;
- possibilità di estendere più facilmente scorciatoie e target.

## Quando usare questo approccio

Questo approccio è molto adatto quando:

- il comando è chiaramente un'azione applicativa;
- esiste più di un punto da cui invocarlo;
- vuoi una UI coerente;
- stai preparando il terreno a una struttura più pulita.

## Quando potrebbe essere eccessivo

Se hai un bottone locale e isolato che fa una cosa minima e irripetibile, introdurre tutto questo può essere troppo. Il punto non è usare sempre i comandi, ma usarli quando portano davvero ordine.

## Idea chiave

Menu e toolbar danno il meglio quando non sono collegati a eventi separati, ma alla **stessa azione logica**.

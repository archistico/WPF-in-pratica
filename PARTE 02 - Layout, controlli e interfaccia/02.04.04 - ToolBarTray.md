# 02.04.04 - `ToolBarTray`

`ToolBarTray` è il contenitore pensato per ospitare una o più `ToolBar`.

Se hai una sola toolbar molto semplice, potresti anche non sentirne subito il bisogno. Ma appena vuoi un'organizzazione un po' più ordinata, `ToolBarTray` diventa il contenitore giusto.

## Esempio base

```xml
<ToolBarTray>
    <ToolBar Header="Principale">
        <Button Content="Nuovo" />
        <Button Content="Apri" />
        <Button Content="Salva" />
    </ToolBar>
</ToolBarTray>
```

## Perché esiste

`ToolBarTray` serve a:

- contenere toolbar in modo corretto;
- gestire più barre strumenti;
- mantenere un'organizzazione più pulita della parte alta della finestra.

## Esempio con più toolbar

```xml
<ToolBarTray DockPanel.Dock="Top">
    <ToolBar Header="File">
        <Button Content="Nuovo" />
        <Button Content="Apri" />
        <Button Content="Salva" />
    </ToolBar>

    <ToolBar Header="Azioni">
        <Button Content="Aggiorna" />
        <Button Content="Esporta" />
    </ToolBar>
</ToolBarTray>
```

Questa soluzione è utile quando vuoi separare gruppi di comandi con significati diversi.

## In che punto della finestra si usa

Di solito viene inserito nella zona superiore del layout, spesso insieme al `Menu`:

```xml
<DockPanel>
    <Menu DockPanel.Dock="Top" />
    <ToolBarTray DockPanel.Dock="Top" />
    <Grid />
</DockPanel>
```

Questo schema è molto classico e ancora oggi molto efficace nelle applicazioni desktop.

## Quando usarlo davvero

Usa `ToolBarTray` quando:

- hai almeno una toolbar strutturata;
- vuoi lasciare aperta la possibilità di aggiungerne altre;
- vuoi un contenitore pensato apposta per questa funzione.

Se invece hai solo due pulsanti rapidi, può essere più semplice metterli in un normale pannello.

## Errore tipico

Un errore frequente è ignorare `ToolBarTray` e inserire toolbar o pseudo-toolbar dentro pannelli generici senza una vera struttura. Non sempre è sbagliato, ma spesso è il segnale che si sta improvvisando il layout.

## Regola pratica

Se stai costruendo una parte alta della finestra con menu e strumenti rapidi, `ToolBarTray` è quasi sempre il contenitore giusto per la barra strumenti.

# 02.04.07 - `CanExecute`: abilitare e disabilitare bene i comandi

Uno dei vantaggi più belli del sistema dei comandi è `CanExecute`.

Con `CanExecute` non dici solo che cosa deve succedere quando un comando parte. Dici anche **se quel comando è attualmente disponibile**.

## Perché è importante

Senza `CanExecute`, spesso si finisce a scrivere codice manuale per:

- abilitare o disabilitare pulsanti;
- abilitare o disabilitare voci di menu;
- ricordarsi di aggiornare tutto quando cambia lo stato della finestra.

Con i comandi, invece, la logica di disponibilità è centralizzata.

## Esempio pratico

Supponiamo di poter salvare solo se il testo non è vuoto.

### XAML

```xml
<Window.CommandBindings>
    <CommandBinding Command="ApplicationCommands.Save"
                    Executed="Save_Executed"
                    CanExecute="Save_CanExecute" />
</Window.CommandBindings>

<DockPanel>
    <Menu DockPanel.Dock="Top">
        <MenuItem Header="_File">
            <MenuItem Header="_Salva" Command="ApplicationCommands.Save" />
        </MenuItem>
    </Menu>

    <ToolBarTray DockPanel.Dock="Top">
        <ToolBar>
            <Button Content="Salva" Command="ApplicationCommands.Save" />
        </ToolBar>
    </ToolBarTray>

    <TextBox x:Name="txtTitolo" Margin="12" TextChanged="txtTitolo_TextChanged" />
</DockPanel>
```

### C#

```csharp
private void Save_CanExecute(object sender, CanExecuteRoutedEventArgs e)
{
    e.CanExecute = !string.IsNullOrWhiteSpace(txtTitolo.Text);
}

private void Save_Executed(object sender, ExecutedRoutedEventArgs e)
{
    MessageBox.Show("Dati salvati.");
}

private void txtTitolo_TextChanged(object sender, TextChangedEventArgs e)
{
    CommandManager.InvalidateRequerySuggested();
}
```

## Cosa si ottiene

Quando `txtTitolo` è vuoto:

- il `MenuItem` si disabilita;
- il `Button` si disabilita.

Quando il testo torna valido, entrambi si riabilitano.

## Perché è meglio di farlo a mano

Immagina la versione senza comandi:

- dovresti aggiornare manualmente `IsEnabled` del bottone;
- poi anche del menu item;
- poi magari anche di altri punti della UI.

Il rischio di incoerenze aumenta subito.

## Attenzione a un aspetto pratico

A volte serve forzare il ricalcolo di `CanExecute`, come nell'esempio con:

```csharp
CommandManager.InvalidateRequerySuggested();
```

Non è sempre necessario in ogni scenario, ma è importante sapere che esiste, perché spesso spiega perché un comando non si aggiorna come ti aspetti.

## Regola pratica

Se un'azione deve essere disponibile solo in certe condizioni, `CanExecute` è quasi sempre il posto giusto in cui esprimere quella regola.

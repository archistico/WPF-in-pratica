# 02.03.08 - `SelectedValuePath` e `SelectedValue`

In molti casi non ti serve lavorare con l’intero oggetto selezionato. Ti serve solo **una sua proprietà specifica**, per esempio:

- l’ID;
- un codice;
- una chiave;
- una sigla.

Qui entrano in gioco `SelectedValuePath` e `SelectedValue`.

## L’idea di base

- `SelectedItem` restituisce l’oggetto completo.
- `SelectedValuePath` indica quale proprietà estrarre dall’oggetto.
- `SelectedValue` riceve il valore di quella proprietà.

## Esempio concreto

### C#

```csharp
public class Area
{
    public int Id { get; set; }
    public string Descrizione { get; set; } = "";
}

public List<Area> Aree { get; } = new()
{
    new Area { Id = 1, Descrizione = "Grafica" },
    new Area { Id = 2, Descrizione = "Programmazione" },
    new Area { Id = 3, Descrizione = "Casa Editrice" }
};

public int AreaIdSelezionata { get; set; }
```

### XAML

```xml
<ComboBox Width="240"
          ItemsSource="{Binding Aree}"
          DisplayMemberPath="Descrizione"
          SelectedValuePath="Id"
          SelectedValue="{Binding AreaIdSelezionata}" />
```

## Cosa succede qui

- la `ComboBox` mostra `Descrizione`;
- ogni elemento è un oggetto `Area`;
- `SelectedValuePath="Id"` dice di prendere la proprietà `Id`;
- `SelectedValue` riceve quell’ID.

Quindi, se l’utente seleziona “Programmazione”, la proprietà `AreaIdSelezionata` vale `2`.

## Quando questa soluzione è utile

È molto utile quando:

- devi memorizzare solo una chiave;
- il resto dell’oggetto non ti serve in quel punto;
- lavori con entità da database;
- hai un form che salva riferimenti tramite ID.

## Confronto diretto con `SelectedItem`

### `SelectedItem`
Usalo quando:
- ti serve l’oggetto completo;
- vuoi leggere più proprietà;
- il codice successivo lavora direttamente sull’entità scelta.

### `SelectedValue`
Usalo quando:
- ti basta una sola proprietà;
- vuoi ottenere direttamente l’ID o il codice;
- vuoi legare il controllo a una proprietà semplice del ViewModel.

## Esempio con entrambi per capire la differenza

```xml
<ComboBox Width="240"
          ItemsSource="{Binding Aree}"
          DisplayMemberPath="Descrizione"
          SelectedItem="{Binding AreaSelezionata}"
          SelectedValuePath="Id"
          SelectedValue="{Binding AreaIdSelezionata}" />
```

Questa configurazione è possibile, ma va usata con criterio. In genere conviene avere chiaro quale informazione vuoi usare come “dato principale” del controllo.

## Quando può essere una cattiva idea usare `SelectedValue`

Usare `SelectedValue` non è sempre la scelta migliore.

Può essere scomodo quando:

- più avanti ti serviranno altre proprietà dell’oggetto;
- la UI dipende da più dati dell’elemento selezionato;
- vuoi mantenere una logica più aderente al modello di dominio.

In questi casi `SelectedItem` è spesso più naturale.

## Esempio realistico da form

Immagina un form progetto in cui devi salvare solo l’ID dell’area:

```xml
<Grid Margin="16">
    <Grid.ColumnDefinitions>
        <ColumnDefinition Width="120" />
        <ColumnDefinition Width="*" />
    </Grid.ColumnDefinitions>

    <TextBlock Text="Area"
               VerticalAlignment="Center" />
    <ComboBox Grid.Column="1"
              Width="240"
              ItemsSource="{Binding Aree}"
              DisplayMemberPath="Descrizione"
              SelectedValuePath="Id"
              SelectedValue="{Binding AreaId}" />
</Grid>
```

Qui la scelta è perfettamente sensata: la `ComboBox` mostra un testo descrittivo, ma il valore che interessa davvero al salvataggio è l’ID.

## Errore tipico

Un errore molto comune è aspettarsi che `SelectedValue` contenga qualcosa di utile senza aver impostato `SelectedValuePath`.

Per esempio:

```xml
<ComboBox ItemsSource="{Binding Aree}"
          SelectedValue="{Binding AreaId}" />
```

Senza `SelectedValuePath`, il risultato può non essere quello atteso.

## Un altro errore comune

Mescolare `SelectedItem` e `SelectedValue` senza avere un criterio chiaro. Funziona, ma spesso rende più confusa la logica dell’interfaccia.

La domanda giusta è:

> Mi serve l’oggetto completo oppure mi basta una sua proprietà?

## Regola pratica

Se stai costruendo una schermata che deve salvare un riferimento tramite chiave, `SelectedValuePath` + `SelectedValue` è spesso la scelta migliore. Se invece devi lavorare davvero con l’oggetto selezionato, usa `SelectedItem`.

## Il punto da portare a casa

`SelectedValuePath` e `SelectedValue` servono quando la selezione deve restituire **una proprietà specifica** dell’oggetto, non l’oggetto intero. Sono particolarmente utili nei form e nelle schermate collegate a dati persistenti, dove spesso l’informazione importante è l’ID.

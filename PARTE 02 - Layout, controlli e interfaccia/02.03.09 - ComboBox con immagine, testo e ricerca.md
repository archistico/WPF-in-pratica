# 02.03.09 - `ComboBox` con immagine, testo e ricerca

Quando una `ComboBox` deve essere più ricca di una semplice lista di testi, è normale voler mostrare:

- un’icona o un’immagine;
- un testo principale;
- magari un testo secondario;
- una ricerca rapida tramite digitazione.

Questo è uno dei casi più interessanti, perché mette insieme più concetti già visti:

- `ItemsSource`;
- `SelectedItem`;
- `DisplayMemberPath` o `TextSearch`;
- `ItemTemplate`;
- `IsEditable`;
- `IsTextSearchEnabled`.

## Obiettivo del controllo

Immagina una schermata in cui l’utente deve scegliere un prodotto o una categoria visualizzando:

- una piccola immagine;
- il nome;
- eventualmente un codice.

La lista deve anche essere comoda da cercare.

## Classe dati di esempio

```csharp
public class ProdottoBreve
{
    public int Id { get; set; }
    public string Nome { get; set; } = "";
    public string Codice { get; set; } = "";
    public string Immagine { get; set; } = "";
}
```

## Dati di esempio

```csharp
public List<ProdottoBreve> Prodotti { get; } = new()
{
    new ProdottoBreve { Id = 1, Nome = "Mouse", Codice = "HW001", Immagine = "/Images/mouse.png" },
    new ProdottoBreve { Id = 2, Nome = "Tastiera", Codice = "HW002", Immagine = "/Images/tastiera.png" },
    new ProdottoBreve { Id = 3, Nome = "Monitor", Codice = "HW003", Immagine = "/Images/monitor.png" }
};

public ProdottoBreve? ProdottoSelezionato { get; set; }
```

## Esempio XAML

```xml
<ComboBox Width="320"
          IsEditable="True"
          IsTextSearchEnabled="True"
          ItemsSource="{Binding Prodotti}"
          SelectedItem="{Binding ProdottoSelezionato}"
          DisplayMemberPath="Nome">
    <ComboBox.ItemTemplate>
        <DataTemplate>
            <StackPanel Orientation="Horizontal" Margin="2">
                <Image Source="{Binding Immagine}"
                       Width="20"
                       Height="20"
                       Margin="0,0,8,0" />
                <StackPanel>
                    <TextBlock Text="{Binding Nome}"
                               FontWeight="SemiBold" />
                    <TextBlock Text="{Binding Codice}"
                               FontSize="11"
                               Opacity="0.75" />
                </StackPanel>
            </StackPanel>
        </DataTemplate>
    </ComboBox.ItemTemplate>
</ComboBox>
```

## Perché questo esempio è interessante

Qui la `ComboBox` fa più cose insieme:

- visualizza oggetti complessi;
- mostra una struttura grafica ricca nella lista;
- permette una ricerca rapida per nome;
- mantiene una selezione di tipo oggetto.

## Il ruolo di `DisplayMemberPath="Nome"`

Questo dettaglio è importante.

Anche se la lista usa un `ItemTemplate`, `DisplayMemberPath="Nome"` continua a essere utile perché fornisce un riferimento testuale semplice e leggibile. In particolare, in una `ComboBox` editabile, è spesso utile avere una proprietà testuale chiara per la parte scrivibile e per il comportamento di ricerca.

## Quando questa soluzione ha senso

È una buona scelta quando:

- l’elemento va riconosciuto visivamente;
- il solo testo non basta;
- un’icona migliora la comprensione;
- la lista non è minuscola e la digitazione accelera la selezione.

## Quando non conviene

Non conviene quando:

- l’immagine non aggiunge davvero informazione;
- il controllo diventa troppo carico visivamente;
- la dropdown è usata in contesti molto compatti;
- una lista semplice sarebbe più leggibile.

## Un dettaglio importante sulla ricerca

La ricerca rapida qui è utile soprattutto se il testo significativo è chiaro. Se l’utente deve cercare per codice, nome e altri campi insieme, questa soluzione potrebbe non bastare. In quel caso serve un filtro esplicito, che vedremo nel prossimo sottocapitolo.

## Possibile variante: testo principale e testo secondario

Se non vuoi immagini ma vuoi comunque una resa più ricca, puoi semplificare il template:

```xml
<ComboBox Width="300"
          ItemsSource="{Binding Clienti}"
          SelectedItem="{Binding ClienteSelezionato}"
          IsEditable="True"
          IsTextSearchEnabled="True"
          DisplayMemberPath="RagioneSociale">
    <ComboBox.ItemTemplate>
        <DataTemplate>
            <StackPanel>
                <TextBlock Text="{Binding RagioneSociale}" />
                <TextBlock Text="{Binding Citta}" FontSize="11" />
            </StackPanel>
        </DataTemplate>
    </ComboBox.ItemTemplate>
</ComboBox>
```

## Errore tipico

Un errore comune è aggiungere immagini, template, editabilità e ricerca tutti insieme senza chiedersi se il caso lo richiede davvero.

La domanda giusta è sempre:

> Questa ricchezza visuale aiuta davvero l’utente a scegliere meglio?

Se la risposta è no, il controllo sta diventando più pesante del necessario.

## Regola pratica

Usa una `ComboBox` ricca con immagine e testo quando l’elemento deve essere riconosciuto rapidamente e il solo testo non basta. Aggiungi la ricerca rapida solo se la lista è abbastanza lunga da giustificarla.

## Il punto da portare a casa

Una `ComboBox` avanzata può combinare `ItemTemplate`, immagini, testo multiplo e ricerca rapida in modo molto efficace. Ma ogni elemento in più deve avere una motivazione precisa: rendere la scelta più chiara e più veloce, non solo rendere il controllo più appariscente.

# 02.03.10 - `ComboBox` con filtro vero e proprio

Fin qui abbiamo visto la ricerca rapida integrata. È utile, ma non modifica davvero la lista.

Un filtro vero è un’altra cosa: mentre l’utente digita, gli elementi mostrati si riducono realmente ai soli risultati compatibili.

Questo è uno dei casi in cui la `ComboBox` smette di essere un controllo “base” e diventa una piccola interfaccia di ricerca.

## Quando serve davvero un filtro vero

Serve quando:

- la lista è lunga;
- la ricerca rapida non basta;
- vuoi mostrare solo i risultati compatibili;
- vuoi cercare su una o più proprietà;
- vuoi dare una sensazione simile a un autocomplete controllato.

## Idea di fondo

Per ottenere un filtro vero, in WPF di solito si lavora con una vista dei dati, per esempio una `CollectionView`, e si aggiorna il filtro in base al testo digitato.

L’idea è questa:

1. hai una collezione completa;
2. esponi una vista filtrabile;
3. mentre l’utente digita, aggiorni il criterio di filtro;
4. la `ComboBox` mostra la vista filtrata.

## Esempio di classe dati

```csharp
public class Cliente
{
    public int Id { get; set; }
    public string RagioneSociale { get; set; } = "";
    public string Citta { get; set; } = "";
}
```

## Esempio C#

```csharp
using System.Collections.ObjectModel;
using System.ComponentModel;
using System.Windows;
using System.Windows.Data;

public partial class MainWindow : Window, INotifyPropertyChanged
{
    private string _testoFiltro = "";

    public ObservableCollection<Cliente> Clienti { get; } = new()
    {
        new Cliente { Id = 1, RagioneSociale = "Alfa Srl", Citta = "Torino" },
        new Cliente { Id = 2, RagioneSociale = "Beta Spa", Citta = "Milano" },
        new Cliente { Id = 3, RagioneSociale = "Gamma Studio", Citta = "Aosta" },
        new Cliente { Id = 4, RagioneSociale = "Delta SNC", Citta = "Torino" }
    };

    public ICollectionView ClientiView { get; }

    public string TestoFiltro
    {
        get => _testoFiltro;
        set
        {
            if (_testoFiltro != value)
            {
                _testoFiltro = value;
                OnPropertyChanged(nameof(TestoFiltro));
                ClientiView.Refresh();
            }
        }
    }

    public Cliente? ClienteSelezionato { get; set; }

    public MainWindow()
    {
        InitializeComponent();
        DataContext = this;

        ClientiView = CollectionViewSource.GetDefaultView(Clienti);
        ClientiView.Filter = FiltraCliente;
    }

    private bool FiltraCliente(object obj)
    {
        if (obj is not Cliente cliente)
            return false;

        if (string.IsNullOrWhiteSpace(TestoFiltro))
            return true;

        return cliente.RagioneSociale.Contains(TestoFiltro, StringComparison.OrdinalIgnoreCase)
            || cliente.Citta.Contains(TestoFiltro, StringComparison.OrdinalIgnoreCase);
    }

    public event PropertyChangedEventHandler? PropertyChanged;

    private void OnPropertyChanged(string nome)
    {
        PropertyChanged?.Invoke(this, new PropertyChangedEventArgs(nome));
    }
}

public class Cliente
{
    public int Id { get; set; }
    public string RagioneSociale { get; set; } = "";
    public string Citta { get; set; } = "";
}
```

## Esempio XAML

```xml
<StackPanel Margin="16">
    <TextBlock Text="Cerca cliente" Margin="0,0,0,6" />
    <TextBox Width="280"
             Text="{Binding TestoFiltro, UpdateSourceTrigger=PropertyChanged}" />

    <TextBlock Text="Cliente" Margin="0,12,0,6" />
    <ComboBox Width="280"
              ItemsSource="{Binding ClientiView}"
              SelectedItem="{Binding ClienteSelezionato}">
        <ComboBox.ItemTemplate>
            <DataTemplate>
                <StackPanel>
                    <TextBlock Text="{Binding RagioneSociale}" FontWeight="SemiBold" />
                    <TextBlock Text="{Binding Citta}" FontSize="11" />
                </StackPanel>
            </DataTemplate>
        </ComboBox.ItemTemplate>
    </ComboBox>
</StackPanel>
```

## Perché in questo esempio il filtro è “vero”

Perché non stai solo cercando un elemento. Stai cambiando la vista dei dati. La `ComboBox` mostra davvero meno elementi quando il testo del filtro cambia.

## Perché qui uso una `TextBox` separata

Dal punto di vista didattico è la scelta migliore. Così si capisce chiaramente la logica:

- il testo di filtro sta in una proprietà;
- la vista viene aggiornata;
- la `ComboBox` mostra la vista filtrata.

Si può fare anche una soluzione più integrata con `IsEditable=True`, ma è più delicata e meno chiara da introdurre.

## Quando usare questa soluzione invece della sola ricerca rapida

Usala quando:

- la lista è davvero ampia;
- vuoi cercare su più campi;
- vuoi mostrare solo i risultati compatibili;
- l’esperienza di selezione deve essere più guidata.

## Quando è eccessiva

È eccessiva quando:

- l’elenco è piccolo;
- basta `IsTextSearchEnabled`;
- la logica di filtro aggiunge complessità non necessaria;
- la schermata non beneficia davvero di questa interazione.

## Variante concettuale da ricordare

Dal punto di vista pratico, spesso hai due strade:

### Strada 1: ricerca rapida integrata
Più semplice, meno codice, meno controllo.

### Strada 2: filtro vero con vista filtrata
Più codice, più controllo, risultato più forte.

La scelta dipende sempre dal caso d’uso.

## Errore tipico

L’errore più comune è voler costruire subito una `ComboBox` super intelligente anche quando basterebbe una selezione semplice.

Una `ComboBox` filtrata è utile, ma ha senso solo quando il problema lo richiede davvero.

## Il punto da portare a casa

Una `ComboBox` con filtro vero non si limita ad aiutare la selezione: cambia davvero l’elenco mostrato. È la soluzione giusta quando la lista è lunga, la ricerca deve essere seria e il semplice `IsTextSearchEnabled` non basta più.

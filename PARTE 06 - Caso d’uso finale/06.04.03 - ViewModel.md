# 06.04.03 - ViewModel

## Il centro operativo della schermata

Dopo i modelli e il `RelayCommand`, il passo più importante dell'implementazione è il `ViewModel`.

Qui il caso finale prende davvero vita, perché è nel `ViewModel` che la schermata:
- conosce i tavoli;
- conosce i piatti;
- sa qual è il tavolo attivo;
- espone le righe del conto corrente;
- calcola il totale;
- pubblica i comandi dell'operatore.

In altre parole, questo è il vero cuore della cassa del ristorante.

## Quali responsabilità vogliamo dargli

Nel nostro caso finale, il `CassaRistoranteViewModel` deve:
- contenere i dati iniziali;
- esporre collezioni bindabili;
- mantenere la selezione corrente;
- pubblicare proprietà derivate;
- coordinare i comandi.

È importante che questo ruolo sia chiaro, così il lettore vede bene che il view model non è un contenitore casuale di codice, ma il punto che governa la schermata.

## Implementazione

```csharp
using System.Collections.ObjectModel;
using System.ComponentModel;
using System.Linq;
using System.Runtime.CompilerServices;
using System.Windows.Input;
using CassaRistorante.Commands;
using CassaRistorante.Models;

namespace CassaRistorante.ViewModels
{
    public class CassaRistoranteViewModel : INotifyPropertyChanged
    {
        private Tavolo? _tavoloSelezionato;
        private RigaConto? _rigaSelezionata;
        private Piatto? _piattoSelezionato;

        public ObservableCollection<Tavolo> Tavoli { get; } = new();
        public ObservableCollection<Piatto> Piatti { get; } = new();

        public Tavolo? TavoloSelezionato
        {
            get => _tavoloSelezionato;
            set
            {
                if (_tavoloSelezionato != value)
                {
                    _tavoloSelezionato = value;
                    OnPropertyChanged();
                    OnPropertyChanged(nameof(RigheContoCorrenti));
                    OnPropertyChanged(nameof(TotaleCorrente));
                    AggiornaStatoComandi();
                }
            }
        }

        public RigaConto? RigaSelezionata
        {
            get => _rigaSelezionata;
            set
            {
                if (_rigaSelezionata != value)
                {
                    _rigaSelezionata = value;
                    OnPropertyChanged();
                    AggiornaStatoComandi();
                }
            }
        }

        public Piatto? PiattoSelezionato
        {
            get => _piattoSelezionato;
            set
            {
                if (_piattoSelezionato != value)
                {
                    _piattoSelezionato = value;
                    OnPropertyChanged();
                    AggiornaStatoComandi();
                }
            }
        }

        public ObservableCollection<RigaConto> RigheContoCorrenti
            => TavoloSelezionato?.RigheConto ?? new ObservableCollection<RigaConto>();

        public decimal TotaleCorrente
            => TavoloSelezionato?.RigheConto.Sum(r => r.Subtotale) ?? 0m;

        public ICommand AggiungiPiattoCommand { get; }
        public ICommand RimuoviRigaCommand { get; }
        public ICommand AumentaQuantitaCommand { get; }
        public ICommand DiminuisciQuantitaCommand { get; }
        public ICommand ChiudiContoCommand { get; }

        public CassaRistoranteViewModel()
        {
            SeedDatiIniziali();

            AggiungiPiattoCommand = new RelayCommand(
                _ => AggiungiPiatto(),
                _ => TavoloSelezionato is not null && PiattoSelezionato is not null);

            RimuoviRigaCommand = new RelayCommand(
                _ => RimuoviRiga(),
                _ => TavoloSelezionato is not null && RigaSelezionata is not null);

            AumentaQuantitaCommand = new RelayCommand(
                _ => AumentaQuantita(),
                _ => RigaSelezionata is not null);

            DiminuisciQuantitaCommand = new RelayCommand(
                _ => DiminuisciQuantita(),
                _ => RigaSelezionata is not null);

            ChiudiContoCommand = new RelayCommand(
                _ => ChiudiConto(),
                _ => TavoloSelezionato is not null && RigheContoCorrenti.Any());
        }

        private void SeedDatiIniziali()
        {
            Piatti.Add(new Piatto { Id = 1, Nome = "Margherita", Prezzo = 7.50m, Categoria = "Pizze" });
            Piatti.Add(new Piatto { Id = 2, Nome = "Carbonara", Prezzo = 9.00m, Categoria = "Primi" });
            Piatti.Add(new Piatto { Id = 3, Nome = "Tiramisù", Prezzo = 5.00m, Categoria = "Dessert" });
            Piatti.Add(new Piatto { Id = 4, Nome = "Acqua", Prezzo = 2.00m, Categoria = "Bevande" });

            Tavoli.Add(new Tavolo { Id = 1, Nome = "Tavolo 1" });
            Tavoli.Add(new Tavolo { Id = 2, Nome = "Tavolo 2" });
            Tavoli.Add(new Tavolo { Id = 3, Nome = "Tavolo 3" });

            TavoloSelezionato = Tavoli.FirstOrDefault();
        }

        private void AggiungiPiatto()
        {
            if (TavoloSelezionato is null || PiattoSelezionato is null)
                return;

            var rigaEsistente = TavoloSelezionato.RigheConto
                .FirstOrDefault(r => r.Piatto.Id == PiattoSelezionato.Id);

            if (rigaEsistente is not null)
            {
                rigaEsistente.Quantita++;
            }
            else
            {
                TavoloSelezionato.RigheConto.Add(new RigaConto
                {
                    Piatto = PiattoSelezionato,
                    Quantita = 1
                });
            }

            OnPropertyChanged(nameof(RigheContoCorrenti));
            OnPropertyChanged(nameof(TotaleCorrente));
            AggiornaStatoComandi();
        }

        private void RimuoviRiga()
        {
            if (TavoloSelezionato is null || RigaSelezionata is null)
                return;

            TavoloSelezionato.RigheConto.Remove(RigaSelezionata);
            RigaSelezionata = null;

            OnPropertyChanged(nameof(RigheContoCorrenti));
            OnPropertyChanged(nameof(TotaleCorrente));
            AggiornaStatoComandi();
        }

        private void AumentaQuantita()
        {
            if (RigaSelezionata is null)
                return;

            RigaSelezionata.Quantita++;
            OnPropertyChanged(nameof(TotaleCorrente));
            AggiornaStatoComandi();
        }

        private void DiminuisciQuantita()
        {
            if (TavoloSelezionato is null || RigaSelezionata is null)
                return;

            RigaSelezionata.Quantita--;

            if (RigaSelezionata.Quantita <= 0)
            {
                TavoloSelezionato.RigheConto.Remove(RigaSelezionata);
                RigaSelezionata = null;
            }

            OnPropertyChanged(nameof(RigheContoCorrenti));
            OnPropertyChanged(nameof(TotaleCorrente));
            AggiornaStatoComandi();
        }

        private void ChiudiConto()
        {
            if (TavoloSelezionato is null)
                return;

            TavoloSelezionato.RigheConto.Clear();
            RigaSelezionata = null;

            OnPropertyChanged(nameof(RigheContoCorrenti));
            OnPropertyChanged(nameof(TotaleCorrente));
            AggiornaStatoComandi();
        }

        private void AggiornaStatoComandi()
        {
            (AggiungiPiattoCommand as RelayCommand)?.RaiseCanExecuteChanged();
            (RimuoviRigaCommand as RelayCommand)?.RaiseCanExecuteChanged();
            (AumentaQuantitaCommand as RelayCommand)?.RaiseCanExecuteChanged();
            (DiminuisciQuantitaCommand as RelayCommand)?.RaiseCanExecuteChanged();
            (ChiudiContoCommand as RelayCommand)?.RaiseCanExecuteChanged();
        }

        public event PropertyChangedEventHandler? PropertyChanged;

        protected void OnPropertyChanged([CallerMemberName] string? propertyName = null)
        {
            PropertyChanged?.Invoke(this, new PropertyChangedEventArgs(propertyName));
        }
    }
}
```

## Perché questa implementazione è didatticamente molto buona

Perché fa vedere tutto in un solo punto:
- stato della schermata;
- dati iniziali;
- selezione;
- comandi;
- proprietà derivate;
- rapporto tra operazioni e aggiornamento UI.

È esattamente il tipo di view model finale che aiuta il lettore a vedere come i concetti studiati nel libro si compongono davvero.

## Un punto molto importante su `RigheContoCorrenti`

Questa proprietà non contiene una collection propria indipendente.
Espone invece la collection del tavolo selezionato.

È una scelta molto interessante, perché:
- è naturale;
- segue il dominio reale;
- rende chiarissimo il rapporto tra contesto corrente e dati mostrati.

L'importante è notificare il cambio del tavolo selezionato, così la UI aggiorna bene il contenuto visibile.

## Un punto importante su `TotaleCorrente`

Anche questa è una proprietà derivata.

Non viene memorizzata come numero che aggiorni manualmente in cento punti.
Viene calcolata dal conto corrente.

Questa è una scelta molto WPF-friendly e molto sana, perché riduce il rischio di incoerenza.

## Che cosa stiamo ancora tenendo intenzionalmente fuori

Per mantenere il caso finale leggibile, qui non stiamo ancora introducendo:
- persistenza SQLite;
- servizi applicativi;
- separazione in più livelli;
- gestione avanzata degli errori;
- validazioni complesse.

È una scelta voluta.
Il focus è mostrare una implementazione pulita della schermata, non costruire un prodotto enterprise completo.

## Regola pratica

Nel caso finale, il `ViewModel` deve fare da centro di gravità della schermata:
espone lo stato, orchestra le azioni e rende naturale il binding della UI.
Se il lettore lo percepisce così, il caso d'uso ha davvero compiuto il suo lavoro.

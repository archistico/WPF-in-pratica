# 04.04.12 - Feedback visivo

## Il drag and drop è molto migliore quando l'utente capisce che cosa sta succedendo

Un drag and drop tecnicamente corretto può comunque risultare scomodo se l'utente non riceve un buon feedback visivo.

Per questo il feedback conta molto.
Serve a chiarire:
- se il target è valido;
- che effetto avrà il drop;
- dove il dato può essere rilasciato;
- quando il puntatore entra o esce da un'area utile.

In altre parole, il feedback visivo rende il drag and drop leggibile.

## Il primo livello di feedback: il cursore

Il sistema di effetti (`Copy`, `Move`, `None`) influisce sul cursore e comunica già molte informazioni.

Per esempio:
- `None` suggerisce che il target non accetta;
- `Copy` e `Move` suggeriscono che il drop è valido.

Questo è il primo livello di feedback e spesso è già molto utile.

## Il secondo livello: evidenziare il target

Molto spesso conviene evidenziare il controllo o il contenitore che può ricevere il drop.

Per esempio:
- cambiare il bordo;
- cambiare lo sfondo;
- mostrare una lieve ombreggiatura;
- far comparire un messaggio guida.

Questo è molto utile soprattutto quando l'area di drop non è ovvia.

## Un esempio semplice

XAML:

```xml
<Border x:Name="dropArea"
        AllowDrop="True"
        DragEnter="DropArea_DragEnter"
        DragLeave="DropArea_DragLeave"
        DragOver="DropArea_DragOver"
        Drop="DropArea_Drop"
        BorderThickness="2"
        BorderBrush="Gray"
        Padding="20">
    <TextBlock Text="Trascina qui i file" />
</Border>
```

Code-behind:

```csharp
private void DropArea_DragEnter(object sender, DragEventArgs e)
{
    dropArea.BorderBrush = Brushes.DodgerBlue;
}

private void DropArea_DragLeave(object sender, DragEventArgs e)
{
    dropArea.BorderBrush = Brushes.Gray;
}

private void DropArea_DragOver(object sender, DragEventArgs e)
{
    e.Effects = e.Data.GetDataPresent(DataFormats.FileDrop)
        ? DragDropEffects.Copy
        : DragDropEffects.None;

    e.Handled = true;
}

private void DropArea_Drop(object sender, DragEventArgs e)
{
    dropArea.BorderBrush = Brushes.Gray;

    if (e.Data.GetDataPresent(DataFormats.FileDrop))
    {
        var files = (string[])e.Data.GetData(DataFormats.FileDrop)!;
        // gestione file
    }
}
```

## Che cosa mostra questo esempio

### `DragEnter`
Evidenzia il target.

### `DragLeave`
Ripristina l'aspetto normale.

### `DragOver`
Decide l'effetto e quindi anche il feedback del cursore.

### `Drop`
Conclude l'operazione e pulisce il feedback.

Questo schema è molto sano e molto leggibile.

## Perché il feedback visivo è importante in app reali

Perché in app desktop vere l'utente deve capire rapidamente:
- se sta trascinando nel posto giusto;
- che cosa succederà se rilascia;
- perché a volte un drop viene rifiutato.

Senza feedback, il drag and drop sembra capriccioso.
Con feedback, sembra controllato e affidabile.

## Un principio molto utile

Il feedback visivo non dovrebbe essere rumoroso o teatrale.
Dovrebbe essere:
- chiaro;
- sobrio;
- coerente;
- immediato.

Questo è particolarmente importante in applicazioni gestionali o professionali.

## Un caso ancora più utile

Nei `UserControl` o in aree di importazione file, può essere molto utile mostrare:
- un testo diverso durante il drag;
- un'icona di drop attivo;
- un bordo più evidente finché il puntatore resta sopra il target.

Piccoli dettagli come questi migliorano molto l'usabilità.

## Errore tipico

L'errore classico è affidarsi solo al cursore, anche quando l'area di drop è poco evidente.
Oppure, al contrario, usare effetti visivi troppo vistosi che distraggono.

## Regola pratica

Nel drag and drop, il feedback visivo serve a rendere chiaro:
- se il target è valido;
- dove si può rilasciare;
- che effetto avrà il drop.
Se l'utente lo capisce a colpo d'occhio, il feedback sta funzionando bene.

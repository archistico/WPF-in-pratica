# 04.03.08 - Inserimento

## L'operazione più istruttiva per capire il doppio livello

L'inserimento è una delle operazioni che mostrano meglio il doppio livello del CRUD con binding:

- devi inserire il dato nel database;
- devi mantenere coerente la raccolta bindata.

Se uno dei due livelli viene dimenticato, il comportamento risulta incompleto.

## Il flusso corretto dell'inserimento

Quando l'utente aggiunge un nuovo record, il flusso corretto è:

1. crei un oggetto `Prodotto`;
2. chiedi al repository di salvarlo in SQLite;
3. recuperi l'`Id` generato;
4. aggiorni l'oggetto in memoria;
5. aggiungi l'oggetto alla `ObservableCollection`.

Questa sequenza è molto importante da capire bene.

## Un esempio nel view model

```csharp
public void AddProdotto(string nome, decimal prezzo)
{
    var nuovo = new Prodotto
    {
        Nome = nome,
        Prezzo = prezzo
    };

    nuovo.Id = _repository.Insert(nuovo);

    Prodotti.Add(nuovo);
    ProdottoSelezionato = nuovo;
}
```

## Come leggere questo metodo

### 1. Crei l'oggetto
Il nuovo prodotto esiste in memoria.

### 2. Lo inserisci nel database
Il repository esegue l'`INSERT`.

### 3. Ricevi l'`Id`
Questo tiene allineato l'oggetto .NET con la riga SQLite.

### 4. Lo aggiungi alla raccolta bindata
La UI vede il nuovo elemento.

Questo è il pattern corretto.

## Perché non basta aggiungere alla collezione

Se fai solo:

```csharp
Prodotti.Add(nuovo);
```

la UI si aggiorna, ma il dato non è persistito.

Se chiudi e riapri l'app, il record non c'è.

## Perché non basta inserire nel database

Se fai solo il salvataggio nel repository ma non aggiorni la raccolta bindata, il database è corretto ma la UI non mostra subito il nuovo record.
Questo crea una schermata incoerente o costringe a reload inutili.

## Due strategie possibili dopo l'inserimento

### 1. Aggiunta diretta alla raccolta
È molto efficiente e molto chiara.

### 2. Reload completo da database
Più semplice da ragionare in certi casi, ma un po' più pesante.

Per un CRUD didattico ben progettato, l'aggiunta diretta è molto istruttiva e del tutto valida.

## Un esempio con comando

```csharp
public ICommand AggiungiCommand { get; }

public ProdottiViewModel()
{
    _repository.EnsureDatabase();
    LoadData();

    AggiungiCommand = new RelayCommand(_ => AddProdotto("Nuovo prodotto", 0m));
}
```

Questo ti permette di collegare l'inserimento direttamente alla UI in modo moderno.

## Un criterio utile

L'inserimento è spesso il punto in cui si vede se il lettore ha capito bene il modello mentale:
- repository per la persistenza;
- collection per la UI;
- view model per l'orchestrazione.

Se questo passaggio è chiaro, il resto del CRUD risulta molto più naturale.

## Errore tipico

L'errore classico è aggiornare solo uno dei due mondi:
- solo la UI;
- oppure solo il database.

Il CRUD corretto deve mantenere coerenti entrambi.

## Regola pratica

Quando inserisci un nuovo elemento in un CRUD WPF con binding, non devi pensare solo “aggiungi riga”.
Devi pensare:
- persisti il dato;
- aggiorna la raccolta bindata.
Entrambi i passaggi contano.

# 02.05.01 - `DatePicker`

## A cosa serve davvero

`DatePicker` serve a permettere all'utente di scegliere una data senza doverla digitare a mano in modo libero. È utile quando vuoi ridurre errori di formato, evitare input ambigui e rendere più rapida la compilazione di form.

In un'applicazione reale lo trovi spesso in:
- filtri per intervallo di date;
- schede anagrafiche;
- scadenze, appuntamenti, prenotazioni;
- maschere di ricerca e report.

Il vantaggio rispetto a una semplice `TextBox` è evidente: con una `TextBox` devi validare il formato, gestire date non valide e interpretare correttamente ciò che l'utente ha scritto. Con `DatePicker` parti già da un controllo pensato per una data.

## Le proprietà da conoscere

Le proprietà più importanti sono:

- `SelectedDate`: la data selezionata;
- `DisplayDate`: la data che il calendario mostra inizialmente;
- `SelectedDateFormat`: formato di visualizzazione;
- `IsDropDownOpen`: indica se il calendario è aperto;
- `DisplayDateStart` e `DisplayDateEnd`: limitano il range consentito.

Esempio semplice:

```xml
<DatePicker SelectedDate="{Binding DataNascita}"
            SelectedDateFormat="Short" />
```

Qui il controllo mostra una data breve e la collega alla proprietà `DataNascita`.

## Un esempio completo

```xml
<StackPanel Margin="20">
    <TextBlock Text="Data inizio" Margin="0,0,0,6"/>
    <DatePicker SelectedDate="{Binding DataInizio}"
                DisplayDateStart="2020-01-01"
                DisplayDateEnd="2030-12-31"
                SelectedDateFormat="Short"/>

    <TextBlock Margin="0,12,0,6"
               Text="{Binding DataInizio, StringFormat=Hai scelto: {0:dd/MM/yyyy}}"/>
</StackPanel>
```

Questo esempio è interessante perché mostra tre cose importanti:
1. il valore vero sta in `SelectedDate`;
2. puoi limitare le date selezionabili;
3. il binding ti permette di riusare il valore subito nella UI.

## Quando usare `DatePicker`

Usalo quando:
- devi acquisire una singola data;
- vuoi diminuire gli errori di digitazione;
- il contesto è da form o filtro;
- la scelta tramite calendario è più naturale della digitazione libera.

## Quando non è la scelta migliore

Non è l'ideale quando:
- ti serve selezionare un intervallo molto articolato;
- devi mostrare molte date insieme;
- il focus non è sull'inserimento ma sulla consultazione di un calendario completo;
- vuoi una UX molto specializzata, per esempio selezione di mesi, settimane o fasce orarie.

In quei casi `Calendar` o un controllo personalizzato possono essere più adatti.

## Errori tipici

### Trattarlo come una `TextBox`
Molti principianti pensano alla parte testuale visibile del controllo e dimenticano che il dato vero è `SelectedDate`.

### Non gestire i valori null
`SelectedDate` può essere nullo. Se il campo non è obbligatorio, il codice deve considerarlo.

```csharp
if (dpScadenza.SelectedDate.HasValue)
{
    DateTime scadenza = dpScadenza.SelectedDate.Value;
}
```

### Usare `DatePicker` per scenari troppo complessi
Se devi mostrare una visione estesa del mese, degli eventi o di più date insieme, il controllo giusto spesso non è questo.

## Regola pratica

Se devi far scegliere **una data** in modo rapido e sicuro, `DatePicker` è quasi sempre la scelta corretta.
Se invece devi **visualizzare o gestire un calendario**, comincia a ragionare su `Calendar`.

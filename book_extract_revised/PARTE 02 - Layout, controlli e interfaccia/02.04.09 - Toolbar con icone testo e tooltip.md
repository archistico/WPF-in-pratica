# 02.04.09 - Toolbar con icone, testo e tooltip

Una toolbar ben fatta non mostra solo pulsanti. Mostra **azioni riconoscibili rapidamente**.

Per questo, in una `ToolBar`, spesso conviene combinare:

- icona;
- testo;
- tooltip.

## Perché non basta il solo testo

Il solo testo è chiaro ma occupa spazio.

## Perché non basta la sola icona

La sola icona è compatta, ma non sempre è immediata. Se l'icona non è universalmente chiara, l'utente deve indovinare.

La combinazione giusta dipende dal contesto:

- icona + tooltip per toolbar compatte;
- icona + testo per toolbar più descrittive;
- icona + testo + tooltip per azioni importanti o ambigue.

## Esempio con stack interno al bottone

```xml
<ToolBar>
    <Button ToolTip="Crea un nuovo documento">
        <StackPanel Orientation="Horizontal">
            <Rectangle Width="16" Height="16" Margin="0,0,6,0" Fill="SteelBlue" />
            <TextBlock Text="Nuovo" />
        </StackPanel>
    </Button>

    <Button ToolTip="Salva il documento corrente">
        <StackPanel Orientation="Horizontal">
            <Rectangle Width="16" Height="16" Margin="0,0,6,0" Fill="DarkGreen" />
            <TextBlock Text="Salva" />
        </StackPanel>
    </Button>
</ToolBar>
```

Qui uso `Rectangle` solo per simulare piccole icone senza introdurre immagini esterne. In un progetto reale potresti usare:

- immagini raster;
- vettoriali;
- `Path`;
- librerie di icone;
- font iconici.

## Perché i tooltip sono importanti

Il tooltip è prezioso quando:

- il bottone è molto compatto;
- l'icona non è ovvia;
- vuoi spiegare meglio l'azione;
- vuoi aggiungere anche la scorciatoia da tastiera.

Per esempio:

```xml
<Button ToolTip="Salva (Ctrl+S)" />
```

## Un esempio più completo

```xml
<ToolBar>
    <Button Width="90" ToolTip="Nuovo documento (Ctrl+N)">
        <StackPanel Orientation="Horizontal">
            <Rectangle Width="16" Height="16" Margin="0,0,6,0" Fill="SteelBlue" />
            <TextBlock Text="Nuovo" />
        </StackPanel>
    </Button>

    <Button Width="90" ToolTip="Apri documento (Ctrl+O)">
        <StackPanel Orientation="Horizontal">
            <Rectangle Width="16" Height="16" Margin="0,0,6,0" Fill="Goldenrod" />
            <TextBlock Text="Apri" />
        </StackPanel>
    </Button>

    <Button Width="90" ToolTip="Salva documento (Ctrl+S)">
        <StackPanel Orientation="Horizontal">
            <Rectangle Width="16" Height="16" Margin="0,0,6,0" Fill="DarkGreen" />
            <TextBlock Text="Salva" />
        </StackPanel>
    </Button>
</ToolBar>
```

## Regola pratica

Una toolbar buona deve rendere le azioni **veloci da vedere** e **facili da capire**. Se le icone da sole non bastano, aggiungi testo o tooltip. Non costringere l'utente a interpretare simboli ambigui.

# 02.04.11 - Menu, toolbar e MVVM

Finché lavori con esempi iniziali, puoi usare il commanding classico di WPF con `CommandBinding` nel code-behind. È utile e formativo, perché fa capire molto bene la logica di `Command`, `Executed` e `CanExecute`.

Ma quando l'app cresce, entra in scena MVVM.

## Il cambio di prospettiva

Con MVVM la vista non dovrebbe più contenere tutta la logica delle azioni. Dovrebbe invece esporre controlli collegati a proprietà e comandi del `ViewModel`.

Questo vale anche per menu e toolbar.

## Esempio tipico con `ICommand`

### XAML

```xml
<DockPanel>
    <Menu DockPanel.Dock="Top">
        <MenuItem Header="_File">
            <MenuItem Header="_Salva" Command="{Binding SalvaCommand}" />
        </MenuItem>
    </Menu>

    <ToolBarTray DockPanel.Dock="Top">
        <ToolBar>
            <Button Content="Salva" Command="{Binding SalvaCommand}" />
        </ToolBar>
    </ToolBarTray>
</DockPanel>
```

### C# del ViewModel

```csharp
public ICommand SalvaCommand { get; }

public MainViewModel()
{
    SalvaCommand = new RelayCommand(Salva, CanSalva);
}

private void Salva()
{
    // logica di salvataggio
}

private bool CanSalva()
{
    return DocumentoCorrente != null && DocumentoCorrente.Modificato;
}
```

## Perché questo approccio è potente

Qui:

- il menu e la toolbar restano semplici;
- la logica dell'azione vive nel `ViewModel`;
- la vista non deve sapere come si salva davvero;
- testabilità e manutenzione migliorano molto.

## Quando restare sul commanding classico

Resta pure sul commanding classico di WPF quando:

- stai imparando;
- la finestra è piccola;
- il code-behind è ancora accettabile;
- vuoi capire bene il modello originale di WPF.

## Quando passare a MVVM

Passa a MVVM quando:

- le finestre crescono;
- i comandi aumentano;
- la logica UI diventa significativa;
- vuoi separare meglio presentazione e comportamento.

## Idea chiave

Menu e toolbar sono ottimi punti per vedere MVVM in azione, perché mostrano benissimo il vantaggio di collegare più sorgenti della UI allo stesso comando del `ViewModel`.

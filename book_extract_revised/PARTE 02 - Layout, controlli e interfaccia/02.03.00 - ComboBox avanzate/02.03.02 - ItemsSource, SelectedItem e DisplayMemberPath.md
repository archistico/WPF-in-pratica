# 02.03.02 - `ItemsSource`, `SelectedItem` e `DisplayMemberPath`

Queste tre proprietà sono il cuore del funzionamento reale di una `ComboBox` in WPF.

- `ItemsSource` dice **da dove arrivano i dati**.
- `SelectedItem` dice **quale oggetto è selezionato**.
- `DisplayMemberPath` dice **quale proprietà dell’oggetto mostrare**.

Se capisci bene queste tre proprietà, hai già compreso gran parte dell’uso pratico della `ComboBox`.

## Il modello mentale corretto

Molti principianti immaginano la `ComboBox` così:

> “Ho una lista di stringhe e ne scelgo una.”

In WPF il modello corretto è più spesso questo:

> “Ho una collezione di oggetti, la `ComboBox` li visualizza, e uno di questi oggetti diventa selezionato.”

Questo cambia molto il modo di ragionare.

## Esempio di classe dati

```csharp
public class Area
{
    public int Id { get; set; }
    public string Descrizione { get; set; } = "";
}
```

## Esempio di proprietà nel contesto dati

```csharp
public List<Area> Aree { get; } = new()
{
    new Area { Id = 1, Descrizione = "Grafica" },
    new Area { Id = 2, Descrizione = "Programmazione" },
    new Area { Id = 3, Descrizione = "Amministrazione" }
};

public Area? AreaSelezionata { get; set; }
```

## Esempio XAML

```xml
<ComboBox Width="240"
          ItemsSource="{Binding Aree}"
          SelectedItem="{Binding AreaSelezionata}"
          DisplayMemberPath="Descrizione" />
```

## Cosa fa ogni proprietà

### `ItemsSource`

`ItemsSource` collega il controllo a una collezione.

```xml
ItemsSource="{Binding Aree}"
```

Qui la `ComboBox` riceve la lista degli oggetti `Area`.

Senza `ItemsSource`, se non stai scrivendo gli elementi a mano, la lista rimane vuota.

### `SelectedItem`

`SelectedItem` collega il controllo all’oggetto scelto.

```xml
SelectedItem="{Binding AreaSelezionata}"
```

Quando l’utente seleziona “Programmazione”, la proprietà `AreaSelezionata` contiene proprio quell’oggetto `Area`.

Questo è molto comodo quando ti servono più informazioni dell’elemento selezionato, non solo il testo mostrato.

### `DisplayMemberPath`

`DisplayMemberPath` dice quale proprietà visualizzare.

```xml
DisplayMemberPath="Descrizione"
```

La `ComboBox` contiene oggetti `Area`, ma l’utente deve vedere un testo leggibile. Qui mostriamo `Descrizione`.

## Perché queste tre proprietà lavorano bene insieme

In questo esempio:

- `ItemsSource` porta la lista;
- `DisplayMemberPath` decide cosa mostrare;
- `SelectedItem` conserva l’oggetto scelto.

È una configurazione molto pulita e, per i casi semplici, spesso è già sufficiente.

## Esempio completo con finestra

### XAML

```xml
<StackPanel Margin="16">
    <TextBlock Text="Area" Margin="0,0,0,6" />
    <ComboBox Width="240"
              ItemsSource="{Binding Aree}"
              SelectedItem="{Binding AreaSelezionata}"
              DisplayMemberPath="Descrizione" />

    <TextBlock Margin="0,12,0,0"
               Text="{Binding AreaSelezionata.Descrizione}" />
</StackPanel>
```

### C#

```csharp
public partial class MainWindow : Window
{
    public List<Area> Aree { get; } = new()
    {
        new Area { Id = 1, Descrizione = "Grafica" },
        new Area { Id = 2, Descrizione = "Programmazione" },
        new Area { Id = 3, Descrizione = "Casa Editrice" }
    };

    public Area? AreaSelezionata { get; set; }

    public MainWindow()
    {
        InitializeComponent();
        DataContext = this;
        AreaSelezionata = Aree[0];
    }
}

public class Area
{
    public int Id { get; set; }
    public string Descrizione { get; set; } = "";
}
```

## Cosa si impara da questo esempio

Questo esempio mostra una cosa importante: la `ComboBox` non restituisce una stringa, ma un oggetto. Il testo mostrato è solo una vista dell’oggetto.

Questo approccio è utile quando, dopo la selezione, ti servono anche:

- l’ID;
- altre proprietà descrittive;
- flag;
- dati collegati.

## Quando questa configurazione basta davvero

Questa triade è perfetta quando:

- devi mostrare una sola proprietà testuale;
- vuoi ottenere l’oggetto selezionato;
- la rappresentazione visiva è semplice.

Se invece vuoi mostrare più campi o un layout più ricco, `DisplayMemberPath` non basta più e serve un `ItemTemplate`.

## Errore tipico

Un errore molto comune è usare `ItemsSource` e `SelectedItem`, ma dimenticare `DisplayMemberPath` quando gli oggetti non hanno una rappresentazione testuale utile. In quel caso la `ComboBox` può mostrare il nome della classe, per esempio:

```text
MioProgetto.Models.Area
```

È un comportamento corretto dal punto di vista del framework, ma ovviamente sbagliato dal punto di vista dell’interfaccia.

## Il punto da portare a casa

`ItemsSource`, `SelectedItem` e `DisplayMemberPath` rappresentano la configurazione fondamentale di una `ComboBox` moderna in WPF. La lista arriva da una collezione, la selezione restituisce un oggetto, e il testo mostrato è solo una proprietà scelta dell’oggetto stesso.

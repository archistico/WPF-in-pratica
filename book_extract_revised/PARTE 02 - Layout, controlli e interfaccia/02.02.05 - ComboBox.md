# 02.02.05 - `ComboBox`

`ComboBox` è uno dei controlli più usati nei form e nelle schermate gestionali. Serve a permettere una scelta tra più valori senza occupare tutto lo spazio che richiederebbe una lista sempre aperta.

È un controllo molto comune, ma proprio per questo spesso viene usato anche quando non sarebbe la scelta migliore.

## Quando usare `ComboBox`

Usa `ComboBox` quando:

- l’utente deve scegliere un valore tra opzioni note;
- non vuoi mostrare sempre l’intero elenco;
- l’elenco non è enorme ma nemmeno banale;
- il valore scelto è uno solo.

Esempi tipici:

- stato documento;
- area di un progetto;
- tipo cliente;
- categoria prodotto.

## Esempio semplice statico

```xml
<ComboBox Width="180" SelectedIndex="0">
    <ComboBoxItem Content="Da iniziare" />
    <ComboBoxItem Content="In corso" />
    <ComboBoxItem Content="Completato" />
</ComboBox>
```

Questo esempio è utile per capire la sintassi, ma non è il modello più adatto per applicazioni reali.

## Perché l’esempio statico va bene solo all’inizio

Scrivere a mano i `ComboBoxItem` è comodo per test veloci o demo piccole. Appena i dati diventano dinamici, il controllo dovrebbe quasi sempre essere alimentato da una collezione tramite binding.

## Esempio più realistico con binding

```xml
<ComboBox ItemsSource="{Binding Stati}"
          SelectedItem="{Binding StatoSelezionato}"
          Width="220" />
```

Qui il controllo esprime molto meglio il modello WPF:

- i dati arrivano da una sorgente;
- la UI mostra la collezione;
- la scelta viene riflessa in una proprietà del ViewModel o dell’oggetto collegato.

## `SelectedItem` o `SelectedValue`?

Questa è una delle prime domande importanti.

### Usa `SelectedItem` quando:
- vuoi lavorare con l’oggetto intero selezionato;
- ti interessa l’elemento completo.

### Usa `SelectedValue` quando:
- vuoi ricavare solo una proprietà specifica dell’elemento;
- per esempio un ID.

Esempio:

```xml
<ComboBox ItemsSource="{Binding Aree}"
          DisplayMemberPath="Descrizione"
          SelectedValuePath="Id"
          SelectedValue="{Binding AreaIdSelezionata}"
          Width="220" />
```

## Perché qui questa soluzione è utile

Se l’applicazione deve memorizzare l’ID dell’area selezionata, non sempre hai bisogno di portarti dietro l’intero oggetto scelto. In questi casi `SelectedValue` è spesso più adatto.

## `DisplayMemberPath` o `ItemTemplate`?

Anche qui serve criterio.

### Usa `DisplayMemberPath` quando:
- basta mostrare una sola proprietà;
- vuoi una configurazione semplice e pulita.

### Usa `ItemTemplate` quando:
- vuoi mostrare più informazioni;
- vuoi un layout personalizzato;
- devi aggiungere immagini, tag, più righe o stili specifici.

Esempio con `DisplayMemberPath`:

```xml
<ComboBox ItemsSource="{Binding Clienti}"
          DisplayMemberPath="RagioneSociale"
          SelectedItem="{Binding ClienteSelezionato}"
          Width="260" />
```

Esempio con `ItemTemplate`:

```xml
<ComboBox ItemsSource="{Binding Clienti}"
          SelectedItem="{Binding ClienteSelezionato}"
          Width="300">
    <ComboBox.ItemTemplate>
        <DataTemplate>
            <StackPanel>
                <TextBlock Text="{Binding RagioneSociale}"
                           FontWeight="SemiBold" />
                <TextBlock Text="{Binding Citta}"
                           FontSize="11" />
            </StackPanel>
        </DataTemplate>
    </ComboBox.ItemTemplate>
</ComboBox>
```

## Quando scegliere una soluzione o l’altra

Per dati semplici, `DisplayMemberPath` è più rapido.
Per dati ricchi, `ItemTemplate` è più espressivo.

Non conviene usare un template complesso se ti basta visualizzare una sola stringa.

## `ComboBox` o `RadioButton`?

Se le opzioni sono poche e importanti, i `RadioButton` spesso comunicano meglio.
Se le opzioni sono più numerose o vuoi risparmiare spazio, `ComboBox` è più adatto.

Questa motivazione va sempre esplicitata quando si progetta una UI.

## `ComboBox` o `ListBox`?

- `ComboBox` mostra una sola scelta e nasconde il resto fino all’apertura;
- `ListBox` mostra l’elenco in modo persistente.

Se l’utente deve confrontare le opzioni tra loro, una `ListBox` può essere migliore.
Se deve solo selezionarne una in uno spazio compatto, `ComboBox` vince.

## Errori tipici

### Usare `ComboBox` per scelte troppo piccole e ovvie

Se le opzioni sono solo due e devono essere ben visibili, spesso i `RadioButton` sono più chiari.

### Hardcodare gli elementi anche quando i dati sono dinamici

Funziona, ma rompe il flusso naturale di WPF e rende la UI meno riusabile.

### Non distinguere tra oggetto selezionato e valore selezionato

Questo porta spesso a binding confusi o a proprietà del ViewModel scelte male.

### Fare dropdown troppo ricchi troppo presto

È possibile, ma non sempre è utile. Prima bisogna capire se davvero serve una `ComboBox` avanzata o se basta una versione semplice.

## Un esempio realistico da form

```xml
<Grid Margin="16">
    <Grid.RowDefinitions>
        <RowDefinition Height="Auto" />
        <RowDefinition Height="Auto" />
    </Grid.RowDefinitions>
    <Grid.ColumnDefinitions>
        <ColumnDefinition Width="120" />
        <ColumnDefinition Width="*" />
    </Grid.ColumnDefinitions>

    <TextBlock Grid.Row="0"
               Grid.Column="0"
               Text="Area"
               VerticalAlignment="Center" />
    <ComboBox Grid.Row="0"
              Grid.Column="1"
              Width="240"
              ItemsSource="{Binding Aree}"
              DisplayMemberPath="Descrizione"
              SelectedValuePath="Id"
              SelectedValue="{Binding AreaId}" />

    <TextBlock Grid.Row="1"
               Grid.Column="0"
               Text="Stato"
               VerticalAlignment="Center"
               Margin="0,10,0,0" />
    <ComboBox Grid.Row="1"
              Grid.Column="1"
              Width="240"
              Margin="0,10,0,0"
              ItemsSource="{Binding Stati}"
              DisplayMemberPath="Descrizione"
              SelectedItem="{Binding StatoSelezionato}" />
</Grid>
```

Questo è già un uso molto credibile del controllo in un’applicazione gestionale.

## Il punto da portare a casa

`ComboBox` è ottima quando devi far scegliere un solo valore tra più opzioni in poco spazio. Ma va usata con criterio: devi decidere se lavorare con l’oggetto o con un valore, se basta una proprietà o serve un template, e soprattutto se è davvero il controllo più chiaro per quella scelta.

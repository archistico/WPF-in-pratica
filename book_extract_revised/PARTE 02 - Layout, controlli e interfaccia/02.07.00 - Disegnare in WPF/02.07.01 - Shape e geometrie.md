# 02.07.01 - `Shape` e geometrie

## Disegnare in WPF non significa solo mostrare immagini

Una delle caratteristiche più interessanti di WPF è che l'interfaccia non è fatta solo di controlli classici come `Button`, `TextBox` o `DataGrid`. Può anche disegnare elementi grafici vettoriali in modo nativo.

Questo significa che puoi creare:
- linee;
- rettangoli;
- ellissi;
- poligoni;
- tracciati più complessi;

senza dover necessariamente usare un'immagine esterna.

Qui entrano in gioco le classi derivate da `Shape` e il tema delle geometrie.

## Che cos'è una `Shape`

`Shape` è la base di alcuni elementi grafici pronti all'uso, tra cui:
- `Line`
- `Rectangle`
- `Ellipse`
- `Polygon`
- `Polyline`
- `Path`

Questi elementi fanno parte dell'albero visuale e si comportano come veri elementi WPF:
- possono stare dentro un layout;
- hanno proprietà come `Width`, `Height`, `Margin`, `HorizontalAlignment`;
- possono ricevere stili;
- possono essere colorati con brush;
- possono anche partecipare agli eventi.

Questa è una differenza importante rispetto a un semplice file immagine: una `Shape` non è solo un asset grafico, ma un elemento vivo dell'interfaccia.

## Le proprietà principali

Molte `Shape` condividono proprietà fondamentali:

- `Fill`: riempimento interno;
- `Stroke`: colore del bordo o della linea;
- `StrokeThickness`: spessore del bordo o della linea;
- `Stretch`: come adattare la forma allo spazio disponibile.

Esempio molto semplice:

```xml
<Rectangle Width="120"
           Height="60"
           Fill="LightBlue"
           Stroke="SteelBlue"
           StrokeThickness="2"/>
```

Questo crea un rettangolo disegnato direttamente in XAML, senza immagini esterne.

## Le forme più comuni

### `Line`

Utile per separatori grafici, collegamenti o elementi decorativi semplici.

```xml
<Line X1="0" Y1="0"
      X2="200" Y2="0"
      Stroke="Gray"
      StrokeThickness="2"/>
```

### `Rectangle`

Molto usato per riquadri, indicatori, elementi decorativi o contenitori visivi.

```xml
<Rectangle Width="140"
           Height="40"
           RadiusX="6"
           RadiusY="6"
           Fill="#E8EEF9"
           Stroke="#5B7BD5"/>
```

### `Ellipse`

Serve per cerchi o ovali.

```xml
<Ellipse Width="50"
         Height="50"
         Fill="Orange"/>
```

### `Polygon`

Utile quando devi definire una forma tramite punti.

```xml
<Polygon Points="50,0 100,100 0,100"
         Fill="Gold"
         Stroke="DarkGoldenrod"
         StrokeThickness="2"/>
```

## Il ruolo delle geometrie

Dietro molte forme vettoriali c'è il concetto di **geometria**, cioè una descrizione della forma da disegnare.

All'inizio puoi pensare così:
- le `Shape` sono oggetti “pronti” da usare;
- le geometrie sono descrizioni più astratte della forma.

Questa distinzione diventerà ancora più chiara con `Path` e `Drawing`.

## Un esempio realistico

Immagina una schermata dashboard dove vuoi mostrare un piccolo indicatore di stato.

```xml
<StackPanel Orientation="Horizontal" Margin="12">
    <Ellipse Width="14"
             Height="14"
             Fill="LimeGreen"
             Margin="0,0,8,0"/>
    <TextBlock VerticalAlignment="Center"
               Text="Servizio attivo"/>
</StackPanel>
```

Qui una semplice `Ellipse` è più pratica di un'icona esterna:
- si colora facilmente;
- si adatta al tema;
- pesa poco;
- è immediata da mantenere.

## Quando usare `Shape`

Usa una `Shape` quando:
- hai bisogno di elementi grafici semplici;
- vuoi mantenere un approccio vettoriale;
- vuoi evitare immagini esterne per simboli banali;
- la forma fa parte dell'interfaccia e non solo della decorazione.

## Quando non usarla

Non è la scelta migliore quando:
- la grafica è molto complessa;
- hai già asset grafici ben fatti;
- il contenuto è illustrativo più che strutturale;
- il disegno vettoriale renderebbe il markup troppo complicato.

## Errori tipici

### Usare immagini dove basterebbe una forma semplice
Aumenti il numero di asset da gestire senza vero vantaggio.

### Disegnare tutto a mano in XAML
Quando il disegno è troppo articolato, la leggibilità del file peggiora.

### Confondere forma e layout
La `Shape` è un elemento UI, ma non deve diventare una toppa per problemi di impaginazione.

## Regola pratica

Se ti serve una forma semplice, dinamica e ben integrata nell'interfaccia, `Shape` è una soluzione naturale.
Se invece la grafica è ricca o illustrativa, spesso è meglio un asset esterno o una soluzione più strutturata.

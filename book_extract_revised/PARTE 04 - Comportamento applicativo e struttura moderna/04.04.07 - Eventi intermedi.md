# 04.04.07 - Eventi intermedi

## Il drag and drop non vive solo nel momento finale del drop

Molti esempi introduttivi mostrano solo:
- avvio del drag;
- gestione del `Drop`.

Ma in realtà il drag and drop è molto più leggibile e usabile quando tieni conto anche degli eventi intermedi.

Questi eventi servono a:
- capire se il target può accettare il dato;
- impostare l'effetto corretto;
- fornire feedback visivo;
- gestire l'ingresso e l'uscita del puntatore dall'area target.

## Gli eventi più importanti

I più utili da conoscere sono:

- `DragEnter`
- `DragOver`
- `DragLeave`
- `Drop`

`Drop` è l'evento finale.
Gli altri aiutano a costruire tutto ciò che succede prima.

## `DragEnter`

Scatta quando il drag entra nell'area del target.

È utile per:
- inizializzare feedback visivi;
- verificare subito il tipo di dato;
- preparare lo stato del target.

## `DragOver`

Scatta mentre il drag si muove sopra il target.

È spesso il punto migliore per:
- impostare `e.Effects`;
- confermare o negare la possibilità di drop;
- aggiornare il feedback mentre il puntatore si muove.

Molto spesso è l'evento più importante tra quelli intermedi.

## `DragLeave`

Scatta quando il drag esce dall'area del target.

È molto utile per:
- rimuovere evidenziazioni;
- ripristinare colori o bordi;
- pulire lo stato visivo temporaneo.

## Un esempio tipico

XAML:

```xml
<Border AllowDrop="True"
        DragEnter="Border_DragEnter"
        DragOver="Border_DragOver"
        DragLeave="Border_DragLeave"
        Drop="Border_Drop"
        Padding="20">
    <TextBlock Text="Area di drop"/>
</Border>
```

Code-behind:

```csharp
private void Border_DragEnter(object sender, DragEventArgs e)
{
    if (e.Data.GetDataPresent(DataFormats.FileDrop))
        e.Effects = DragDropEffects.Copy;
    else
        e.Effects = DragDropEffects.None;

    e.Handled = true;
}

private void Border_DragOver(object sender, DragEventArgs e)
{
    if (e.Data.GetDataPresent(DataFormats.FileDrop))
        e.Effects = DragDropEffects.Copy;
    else
        e.Effects = DragDropEffects.None;

    e.Handled = true;
}

private void Border_DragLeave(object sender, DragEventArgs e)
{
    // eventuale reset del feedback visivo
}

private void Border_Drop(object sender, DragEventArgs e)
{
    if (e.Data.GetDataPresent(DataFormats.FileDrop))
    {
        var files = (string[])e.Data.GetData(DataFormats.FileDrop)!;
        // gestione finale
    }
}
```

## Perché questi eventi migliorano molto l'esperienza

Perché evitano che l'utente scopra solo alla fine se il drop era valido.
Un buon drag and drop comunica prima:
- se il target accetta;
- che effetto avrà;
- dove sta entrando il puntatore.

Questo rende l'interazione molto più chiara.

## Quando servono davvero

Servono soprattutto quando:
- il target non accetta ogni tipo di dato;
- vuoi mostrare feedback visivo;
- hai aree multiple di drop;
- l'operazione ha significato diverso a seconda del contesto.

## Un criterio pratico

### `Drop`
Gestisce l'azione finale.

### `DragOver`
Decide se il drop è valido e quale effetto mostrare.

### `DragEnter` e `DragLeave`
Migliorano il feedback e lo stato visivo.

Questa è una buona mappa mentale.

## Errore tipico

L'errore classico è ignorare gli eventi intermedi e poi stupirsi che:
- il cursore non dia feedback corretto;
- il target non comunichi bene l'accettazione;
- l'interazione sembri brusca o poco chiara.

## Regola pratica

Il drag and drop ben fatto non si costruisce solo sul `Drop`.
Gli eventi intermedi servono a rendere l'operazione comprensibile e gradevole prima del rilascio finale.

# 04.02.07 - Come caricare il `UserControl` in un’area della pagina

## Il punto pratico della navigazione interna

Dopo aver visto shell, area contenuto e `DataTemplate`, è il momento di rispondere alla domanda più concreta:
come si carica davvero un `UserControl` in una zona della finestra?

La risposta moderna e pulita, in questo pattern, è:
**non carichi manualmente il `UserControl` dal code-behind; assegni un contenuto al `ContentControl` e lasci che WPF scelga la vista giusta tramite `DataTemplate`.**

Questa frase riassume l'intero meccanismo.

## La struttura base della shell

XAML della shell:

```xml
<Grid>
    <Grid.ColumnDefinitions>
        <ColumnDefinition Width="220"/>
        <ColumnDefinition Width="*"/>
    </Grid.ColumnDefinitions>

    <StackPanel Background="#F4F4F4">
        <Button Content="Dashboard"
                Command="{Binding ShowDashboardCommand}"
                Margin="12,12,12,8"/>

        <Button Content="Clienti"
                Command="{Binding ShowClientiCommand}"
                Margin="12,0,12,8"/>
    </StackPanel>

    <ContentControl Grid.Column="1"
                    Content="{Binding CurrentViewModel}" />
</Grid>
```

Qui il `ContentControl` è l'area della pagina che cambia.

## I `DataTemplate`

```xml
<Window.Resources>
    <DataTemplate DataType="{x:Type vm:DashboardViewModel}">
        <views:DashboardView />
    </DataTemplate>

    <DataTemplate DataType="{x:Type vm:ClientiViewModel}">
        <views:ClientiView />
    </DataTemplate>
</Window.Resources>
```

Questi template dicono a WPF quale `UserControl` usare per ciascun tipo di view model.

## Il view model della shell

```csharp
public class ShellViewModel : INotifyPropertyChanged
{
    private object? _currentViewModel;

    public object? CurrentViewModel
    {
        get => _currentViewModel;
        set
        {
            if (_currentViewModel != value)
            {
                _currentViewModel = value;
                OnPropertyChanged();
            }
        }
    }

    public ICommand ShowDashboardCommand { get; }
    public ICommand ShowClientiCommand { get; }

    public ShellViewModel()
    {
        ShowDashboardCommand = new RelayCommand(_ => CurrentViewModel = new DashboardViewModel());
        ShowClientiCommand = new RelayCommand(_ => CurrentViewModel = new ClientiViewModel());

        CurrentViewModel = new DashboardViewModel();
    }

    public event PropertyChangedEventHandler? PropertyChanged;

    protected void OnPropertyChanged([CallerMemberName] string? propertyName = null)
    {
        PropertyChanged?.Invoke(this, new PropertyChangedEventArgs(propertyName));
    }
}
```

## Che cosa succede davvero

### 1. L'utente preme “Clienti”
Il comando cambia `CurrentViewModel`.

### 2. Il `ContentControl` riceve il nuovo contenuto
La property `Content` cambia.

### 3. WPF guarda il tipo del nuovo contenuto
Se è `ClientiViewModel`, cerca un `DataTemplate` adatto.

### 4. Istanzia la vista corretta
In questo caso `ClientiView`.

### 5. La inserisce nell'area contenuto
E la pagina interna cambia senza che la shell debba istanziare la view a mano.

Questo è il cuore elegante dell'approccio.

## Perché è meglio del caricamento manuale

Un approccio meno pulito potrebbe essere:
- creare il `UserControl` nel code-behind;
- assegnarlo manualmente a un contenitore;
- decidere con if/switch quale view usare.

Funziona, ma è più accoppiato e meno WPF-style.

Il pattern con `ContentControl` + `DataTemplate` è molto più dichiarativo e più manutenibile.

## Quando questo approccio è particolarmente buono

È ottimo quando:
- la shell cambia tra più sezioni;
- vuoi restare in MVVM;
- vuoi aggiungere nuove viste facilmente;
- non vuoi infrastrutture di navigazione troppo pesanti;
- vuoi una UI desktop moderna ma semplice da seguire.

## Errori tipici

### Caricare le view a mano nel code-behind della shell
Si perde la pulizia del pattern.

### Mettere direttamente `UserControl` nel `CurrentViewModel`
Il nome stesso ti segnala che stai mescolando livelli.

### Non usare `DataTemplate`
Così perdi il meccanismo più elegante del modello.

## Regola pratica

Per caricare un `UserControl` in un'area della pagina in modo moderno, assegna il view model corrente al `ContentControl` e lascia che il `DataTemplate` scelga la vista giusta.

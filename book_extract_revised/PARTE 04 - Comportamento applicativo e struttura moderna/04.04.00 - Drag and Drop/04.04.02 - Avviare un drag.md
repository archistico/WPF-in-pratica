# 04.04.02 - Avviare un drag

## Il punto in cui il gesto diventa operazione reale

Il drag and drop comincia davvero nel momento in cui la sorgente avvia il trascinamento.

In WPF, questo avviene tipicamente chiamando:

```csharp
DragDrop.DoDragDrop(...)
```

Questa è la chiamata centrale che trasforma un gesto UI in una vera operazione di drag.

## Il metodo chiave

La forma tipica è:

```csharp
DragDrop.DoDragDrop(dragSource, data, allowedEffects);
```

Dove:
- `dragSource` è il controllo o l'elemento sorgente;
- `data` è il payload da trascinare;
- `allowedEffects` indica gli effetti consentiti, per esempio copia o spostamento.

## Un esempio semplice

Supponiamo di avere una `ListBox` e di voler trascinare l'elemento selezionato.

```csharp
private void ListBox_MouseMove(object sender, MouseEventArgs e)
{
    if (e.LeftButton == MouseButtonState.Pressed)
    {
        var listBox = (ListBox)sender;

        if (listBox.SelectedItem is not null)
        {
            DragDrop.DoDragDrop(
                listBox,
                listBox.SelectedItem,
                DragDropEffects.Move | DragDropEffects.Copy);
        }
    }
}
```

## Come leggere questo esempio

### `MouseMove`
Spesso il drag viene avviato qui, quando il mouse si muove con il pulsante sinistro premuto.

### `SelectedItem`
È l'oggetto che vuoi trascinare.

### `DoDragDrop`
Avvia davvero l'operazione di trascinamento.

### `DragDropEffects`
Dichiara gli effetti consentiti.

## Perché non si avvia sempre direttamente sul `MouseDown`

Potresti essere tentato di avviare il drag al primo `MouseDown`, ma in genere è più naturale aspettare un minimo gesto di trascinamento.
Per questo `MouseMove` con `LeftButton` premuto è una scelta molto comune.

## Il dato trascinato può essere semplice o ricco

Nel parametro `data` puoi mettere:
- una stringa;
- un file;
- un oggetto del modello;
- una struttura personalizzata;
- un `DataObject` più articolato.

Nel capitolo partiremo dai casi più semplici, ma è importante sapere che il payload non è limitato a tipi primitivi.

## Un piccolo miglioramento mentale

Quando avvii un drag, non stai “spostando la `ListBox`”.
Stai iniziando una trattativa tra sorgente e potenziali target attorno a un dato.

Questa è una visione molto utile da tenere.

## Cosa restituisce `DoDragDrop`

Il metodo ritorna un `DragDropEffects` che rappresenta l'effetto finale dell'operazione.
Questo può essere utile per capire se il drop è stato:
- copiato;
- spostato;
- rifiutato.

È un dettaglio importante soprattutto se vuoi aggiornare correttamente le collezioni sorgente dopo un `Move`.

## Quando questo passaggio è più delicato

È più delicato quando:
- vuoi evitare trascinamenti accidentali;
- gli elementi sono selezionabili;
- il controllo ha già altri comportamenti mouse;
- il dato trascinato è complesso.

In quei casi conviene progettare bene la soglia e il momento d'avvio.

## Errore tipico

L'errore classico è avviare il drag troppo presto o senza verificare che ci sia davvero un elemento da trascinare.
Questo porta a UX scomoda e codice fragile.

## Regola pratica

Per avviare un drag in WPF, pensa a questo schema:
- l'utente seleziona o individua un dato;
- inizia un vero gesto di trascinamento;
- la sorgente chiama `DragDrop.DoDragDrop(...)` passando il payload corretto.

# 04.04.08 - Un esempio pratico: trascinare da una `ListBox` a un’altra

## Uno degli esempi più didattici di tutto il capitolo

Se vuoi capire davvero il drag and drop in WPF, uno degli esempi migliori è questo:
trascinare un elemento da una `ListBox` a un'altra.

Perché è così utile?
Perché mette insieme:
- sorgente;
- target;
- oggetto trascinato;
- effetti;
- raccolte bindate.

Ed è anche molto vicino a scenari reali di lavoro.

## Scenario

Hai due elenchi:
- `Disponibili`
- `Selezionati`

L'utente trascina un elemento dalla lista di sinistra a quella di destra.

## Il modello

```csharp
public class Cliente
{
    public string Nome { get; set; } = "";
}
```

## Il view model

```csharp
using System.Collections.ObjectModel;

public class MainViewModel
{
    public ObservableCollection<Cliente> Disponibili { get; } = new()
    {
        new Cliente { Nome = "Anna Rossi" },
        new Cliente { Nome = "Luca Bianchi" },
        new Cliente { Nome = "Marco Verdi" }
    };

    public ObservableCollection<Cliente> Selezionati { get; } = new();
}
```

## Lo XAML

```xml
<Grid Margin="20">
    <Grid.ColumnDefinitions>
        <ColumnDefinition Width="*"/>
        <ColumnDefinition Width="40"/>
        <ColumnDefinition Width="*"/>
    </Grid.ColumnDefinitions>

    <ListBox x:Name="lstDisponibili"
             ItemsSource="{Binding Disponibili}"
             DisplayMemberPath="Nome"
             PreviewMouseMove="LstDisponibili_PreviewMouseMove"/>

    <ListBox x:Name="lstSelezionati"
             Grid.Column="2"
             ItemsSource="{Binding Selezionati}"
             DisplayMemberPath="Nome"
             AllowDrop="True"
             DragOver="LstSelezionati_DragOver"
             Drop="LstSelezionati_Drop"/>
</Grid>
```

## Avvio del drag

```csharp
private void LstDisponibili_PreviewMouseMove(object sender, MouseEventArgs e)
{
    if (e.LeftButton == MouseButtonState.Pressed)
    {
        var listBox = (ListBox)sender;

        if (listBox.SelectedItem is Cliente cliente)
        {
            DragDrop.DoDragDrop(
                listBox,
                cliente,
                DragDropEffects.Move);
        }
    }
}
```

## Preparazione del target

```csharp
private void LstSelezionati_DragOver(object sender, DragEventArgs e)
{
    if (e.Data.GetDataPresent(typeof(Cliente)))
        e.Effects = DragDropEffects.Move;
    else
        e.Effects = DragDropEffects.None;

    e.Handled = true;
}
```

## Drop finale

```csharp
private void LstSelezionati_Drop(object sender, DragEventArgs e)
{
    if (DataContext is not MainViewModel vm)
        return;

    if (e.Data.GetDataPresent(typeof(Cliente)))
    {
        var cliente = (Cliente)e.Data.GetData(typeof(Cliente))!;

        if (!vm.Selezionati.Contains(cliente))
        {
            vm.Selezionati.Add(cliente);
            vm.Disponibili.Remove(cliente);
        }
    }
}
```

## Che cosa mostra davvero questo esempio

### 1. La sorgente trascina un oggetto
L'elemento selezionato della prima `ListBox`.

### 2. Il target verifica il tipo
Accetta solo oggetti `Cliente`.

### 3. Il drop modifica le collezioni
Aggiunge in `Selezionati` e rimuove da `Disponibili`.

### 4. La UI si aggiorna da sola
Grazie alle `ObservableCollection<T>`.

Questo è un esempio didatticamente eccellente perché mostra l'intero flusso.

## Perché è importante il controllo `Contains`

Per evitare duplicati nel target, soprattutto in un esempio di move.
È un dettaglio pratico molto utile.

## Un punto importante sul `Move`

Anche se l'effetto è `Move`, la rimozione dalla sorgente non avviene magicamente.
Sei tu, nel codice, a modificare le collezioni.

Questo è uno dei punti più formativi dell'esempio.

## Varianti possibili

Potresti trasformarlo facilmente in:
- `Copy` invece di `Move`;
- drag bidirezionale;
- drag multiplo;
- drag tra oggetti più complessi.

Ma la struttura base resta la stessa.

## Errore tipico

L'errore classico è aspettarsi che il drag sposti automaticamente l'elemento tra le due `ListBox`.
No: sei tu che devi aggiornare le raccolte bindate.

## Regola pratica

Se vuoi capire bene il drag and drop in WPF, l'esempio `ListBox` → `ListBox` è uno dei migliori:
rende chiarissimo che il drop non sposta controlli, ma oggetti dentro collezioni.

# 01.06.05 - Quando togliere `StartupUri` e usare codice

`StartupUri` è ottima finché l'avvio dell'app è semplice. Ma non sempre basta.

Ci sono casi in cui la finestra iniziale non può essere decisa in modo fisso nel markup. Per esempio:

- devi mostrare una finestra di login;
- devi leggere una configurazione iniziale;
- vuoi aprire una shell diversa in base al ruolo utente;
- devi fare controlli preliminari prima di mostrare la UI principale.

In questi scenari conviene togliere `StartupUri` e gestire l'avvio in codice.

Esempio:

**App.xaml**

```xml
<Application x:Class="EsempioWpf.App"
             xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation"
             xmlns:x="http://schemas.microsoft.com/winfx/2006/xaml"
             Startup="Application_Startup">
</Application>
```

**App.xaml.cs**

```csharp
using System.Windows;

namespace EsempioWpf
{
    public partial class App : Application
    {
        private void Application_Startup(object sender, StartupEventArgs e)
        {
            var finestra = new MainWindow();
            MainWindow = finestra;
            finestra.Show();
        }
    }
}
```

Qui il controllo passa al codice. Questo ti permette di prendere decisioni più intelligenti prima di creare la finestra principale.

Per esempio:

```csharp
private void Application_Startup(object sender, StartupEventArgs e)
{
    bool utenteAutenticato = true;

    Window finestra = utenteAutenticato
        ? new MainWindow()
        : new LoginWindow();

    MainWindow = finestra;
    finestra.Show();
}
```

Non è ancora un'architettura complessa, ma fa vedere subito il vantaggio: l'avvio non è più rigido.

Un errore tipico è togliere `StartupUri` troppo presto, senza averne bisogno. Se il progetto è semplice, spesso è meglio lasciare il caso dichiarativo. Il codice di bootstrap va introdotto quando risolve un problema reale, non solo perché “sembra più professionale”.

La regola pratica è questa: usa `StartupUri` per l'avvio lineare; passa al codice quando la scelta della finestra iniziale o la fase di bootstrap diventano dinamiche.

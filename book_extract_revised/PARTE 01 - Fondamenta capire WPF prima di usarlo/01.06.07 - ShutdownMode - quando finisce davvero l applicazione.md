# 01.06.07 - `ShutdownMode`: quando finisce davvero l'applicazione

Uno degli aspetti meno intuitivi all'inizio è questo: chiudere una finestra non significa sempre chiudere l'intera applicazione.

In WPF questo comportamento si controlla con `ShutdownMode`, una proprietà della classe `Application`.

I valori da conoscere sono tre.

## `OnLastWindowClose`

L'app termina quando si chiude l'ultima finestra aperta.

È un comportamento molto naturale in molte applicazioni desktop.

## `OnMainWindowClose`

L'app termina quando si chiude la `MainWindow`, anche se esistono altre finestre ancora aperte.

Questo è utile quando la finestra principale rappresenta davvero il cuore dell'applicazione.

## `OnExplicitShutdown`

L'app termina solo quando chiami esplicitamente `Shutdown()`.

Questo è il caso più controllato e torna utile in scenari particolari.

Esempio:

```csharp
using System.Windows;

namespace EsempioWpf
{
    public partial class App : Application
    {
        public App()
        {
            ShutdownMode = ShutdownMode.OnMainWindowClose;
        }
    }
}
```

Oppure:

```csharp
Application.Current.Shutdown();
```

quando vuoi chiudere l'app in modo esplicito.

Perché è importante capirlo? Perché nelle applicazioni con più finestre il comportamento di chiusura può sorprenderti. Potresti pensare di aver chiuso “solo una schermata” e invece terminare tutto. Oppure il contrario: chiudi la finestra principale e scopri che l'app è ancora viva.

Errore tipico: non sapere quale `ShutdownMode` sia attivo e perdere tempo a inseguire un comportamento che in realtà è previsto.

La regola pratica è questa:

- applicazione semplice: spesso i valori predefiniti sono sufficienti;
- applicazione con finestre multiple o flussi particolari: verifica sempre in modo esplicito come e quando deve terminare davvero.

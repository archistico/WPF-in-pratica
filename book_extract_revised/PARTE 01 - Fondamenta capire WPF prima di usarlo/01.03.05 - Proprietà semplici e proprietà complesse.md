# 01.03.05 - Proprietà semplici e proprietà complesse

Non tutte le proprietà in XAML si scrivono allo stesso modo. Alcune si esprimono bene con un attributo, altre richiedono una sintassi più esplicita perché il loro valore è un oggetto complesso.

## Proprietà semplici

Quando il valore è breve e facilmente leggibile, si usa di solito un attributo.

```xml
<Button Content="Invia" Margin="10" Width="120" />
```

Questa è la forma più comune nelle interfacce semplici.

## Proprietà complesse

Quando invece una proprietà richiede un oggetto con una sua struttura interna, si usa spesso la property element syntax.

```xml
<Button Content="Invia">
    <Button.Background>
        <LinearGradientBrush>
            <GradientStop Color="LightBlue" Offset="0" />
            <GradientStop Color="SteelBlue" Offset="1" />
        </LinearGradientBrush>
    </Button.Background>
</Button>
```

Qui `Background` non riceve un testo semplice, ma un oggetto `LinearGradientBrush`, che a sua volta contiene altri oggetti.

## Il punto concettuale

In XAML una proprietà può ricevere:

- un valore semplice;
- un oggetto;
- una raccolta di oggetti;
- una markup extension;
- una struttura più articolata.

Per questo non esiste un solo modo di scrivere tutto.

## Errore tipico

Molti principianti pensano che, se una proprietà esiste, allora debba sempre potersi scrivere come attributo. Non è così. Dipende dal tipo di valore richiesto e da quanto quel valore è rappresentabile in forma compatta.

## Cosa portare a casa

Le proprietà semplici stanno bene come attributi. Le proprietà complesse richiedono spesso una sintassi come elemento, perché il loro valore è un oggetto vero e proprio.

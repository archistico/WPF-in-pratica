# 01.06.08 - Un esempio completo e realistico

Mettiamo insieme i concetti visti fin qui in un piccolo esempio realistico. L'obiettivo non è creare un'app complessa, ma mostrare bene la divisione tra livello applicazione e livello finestra.

## `App.xaml`

```xml
<Application x:Class="EsempioWpf.App"
             xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation"
             xmlns:x="http://schemas.microsoft.com/winfx/2006/xaml"
             Startup="Application_Startup">
    <Application.Resources>
        <Style TargetType="Button">
            <Setter Property="Margin" Value="4" />
            <Setter Property="Padding" Value="10,5" />
            <Setter Property="MinWidth" Value="90" />
        </Style>
    </Application.Resources>
</Application>
```

Qui abbiamo:

- avvio gestito via evento `Startup`;
- uno stile globale per tutti i pulsanti.

## `App.xaml.cs`

```csharp
using System.Windows;

namespace EsempioWpf
{
    public partial class App : Application
    {
        private void Application_Startup(object sender, StartupEventArgs e)
        {
            ShutdownMode = ShutdownMode.OnMainWindowClose;

            var finestra = new MainWindow();
            MainWindow = finestra;
            finestra.Show();
        }
    }
}
```

Qui decidiamo:

- come parte l'app;
- quale finestra mostrare;
- quando termina davvero.

## `MainWindow.xaml`

```xml
<Window x:Class="EsempioWpf.MainWindow"
        xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation"
        xmlns:x="http://schemas.microsoft.com/winfx/2006/xaml"
        Title="Gestione Clienti"
        Height="200"
        Width="350">
    <Grid Margin="20">
        <StackPanel>
            <TextBlock Text="Benvenuto nell'applicazione clienti"
                       Margin="0,0,0,12"
                       FontSize="18" />
            <Button Content="Apri anagrafica" />
        </StackPanel>
    </Grid>
</Window>
```

## `MainWindow.xaml.cs`

```csharp
using System.Windows;

namespace EsempioWpf
{
    public partial class MainWindow : Window
    {
        public MainWindow()
        {
            InitializeComponent();
        }
    }
}
```

Questo piccolo esempio è utile perché fa vedere con chiarezza la separazione dei compiti:

- `App.xaml` definisce risorse globali e avvio applicativo;
- `App.xaml.cs` gestisce bootstrap e durata;
- `MainWindow.xaml` definisce la UI della finestra;
- `MainWindow.xaml.cs` contiene il codice della finestra.

È una struttura semplice, ma già sana. Ed è molto meglio capirla ora che scoprirla più tardi quando il progetto sarà più grande e più difficile da riordinare.

# 01.05.07 - Namespace e converter

Uno dei primi casi pratici in cui capire bene namespace e assembly diventa indispensabile è l’uso dei converter.

Un converter è una classe che trasforma un valore in un altro durante il binding. Per esempio, può convertire un `bool` in `Visibility`, oppure un numero in una stringa formattata.

Se hai un converter nel tuo progetto:

```csharp
namespace LibroWpf.Converters
{
    public class BoolToVisibilityConverter
    {
    }
}
```

nello XAML devi prima dichiarare il namespace:

```xml
xmlns:conv="clr-namespace:LibroWpf.Converters"
```

Poi puoi usare il converter nelle risorse:

```xml
<Window.Resources>
    <conv:BoolToVisibilityConverter x:Key="BoolToVisibilityConverter" />
</Window.Resources>
```

A quel punto il converter può essere richiamato nel binding.

## Perché questo esempio è utile

Con i converter si capisce subito che XAML non sta lavorando con “nomi simbolici”, ma con classi vere. Se il namespace è sbagliato, se la classe non è visibile o se l’assembly non è quello corretto, il converter semplicemente non verrà trovato.

## Errore tipico

All’inizio capita spesso di creare la classe converter, ma dimenticare di dichiarare il namespace nello XAML. Un altro errore comune è creare il converter in un progetto separato e dimenticare `assembly=`.

## Cosa portare a casa

I converter sono un ottimo banco di prova per capire namespace e assembly: per usarli in XAML devi fare un mapping corretto verso la classe reale che li implementa.

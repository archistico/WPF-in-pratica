# 01.05.05 - Namespace CLR, namespace XAML e assembly: non sono la stessa cosa

Questo è uno dei punti che crea più confusione all’inizio, quindi conviene fissarlo con chiarezza.

## Namespace CLR

È lo spazio dei nomi delle classi nel mondo .NET e C#. Per esempio:

```csharp
namespace LibroWpf.Views
{
    public partial class SchedaCliente
    {
    }
}
```

Qui `LibroWpf.Views` è un namespace CLR.

## Assembly

È l’output compilato del progetto, per esempio un file `.dll` o `.exe`. Un assembly può contenere uno o più namespace CLR.

## Namespace XAML

È il mapping che usi dentro il markup. Non è un namespace .NET “vero” nel senso classico: è il modo con cui XAML traduce un prefisso in un insieme di tipi reali.

Per esempio:

```xml
xmlns:views="clr-namespace:LibroWpf.Views"
```

Qui `views` è un alias XAML. Non è il namespace CLR della classe: è solo il prefisso che hai scelto per usarlo nel markup.

## La distinzione da ricordare

- `LibroWpf.Views` può essere un namespace CLR;
- `LibroWpf` può essere il nome dell’assembly;
- `views:` può essere il prefisso XAML che usi nei tag.

Sono tre livelli diversi, anche se in pratica spesso sembrano molto vicini.

## Errore tipico

Un errore frequente è pensare che il prefisso XAML debba avere lo stesso nome del namespace CLR o dell’assembly. Non è obbligatorio. Il prefisso è una tua scelta locale nel file XAML.

## Cosa portare a casa

Namespace CLR, namespace XAML e assembly lavorano insieme, ma non sono la stessa cosa. Capire questa distinzione ti evita molti errori quando inizi a usare componenti personalizzati.

# 01.03.01 - Il ruolo del tag radice

Ogni file XAML comincia da un elemento radice. In una finestra WPF questo ruolo è spesso svolto da `Window`, ma può essere anche `Page`, `UserControl` o un altro tipo compatibile con il contesto. Il tag radice è importante perché rappresenta l’oggetto principale che il parser XAML deve creare.

Un esempio classico è questo:

```xml
<Window x:Class="EsempioWpf.MainWindow"
        xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation"
        xmlns:x="http://schemas.microsoft.com/winfx/2006/xaml"
        Title="Esempio"
        Height="250"
        Width="400">
</Window>
```

In questo frammento il tag radice non è solo un contenitore visivo. Sta anche dicendo a WPF quale tipo istanziare, quali namespace usare e quale classe C# è collegata a quel markup.

La prima idea da fissare è questa: il file XAML non è un testo decorativo, ma una descrizione strutturata di oggetti .NET. Il tag radice è il punto da cui parte tutta la costruzione dell’interfaccia.

## Cosa contiene di solito il tag radice

All’inizio troverai quasi sempre:

- il tipo radice, per esempio `Window`;
- `x:Class`, che collega il file XAML alla classe C# associata;
- i namespace XAML principali;
- alcune proprietà iniziali come `Title`, `Height`, `Width` o `StartupLocation`.

Non tutti questi elementi sono obbligatori in ogni file, ma fanno parte della forma più comune con cui un progetto WPF prende vita.

## Errore tipico

Un errore frequente all’inizio è guardare il tag radice come una specie di intestazione “tecnica” da ignorare. In realtà è una parte fondamentale del file. Se è sbagliato il tipo radice, se manca un namespace necessario o se `x:Class` non corrisponde alla classe giusta, il progetto può non compilare oppure il designer può smettere di funzionare correttamente.

## Cosa portare a casa

Il tag radice è il punto di partenza del file XAML. Definisce l’oggetto principale, introduce i namespace necessari e collega il markup al resto dell’applicazione.

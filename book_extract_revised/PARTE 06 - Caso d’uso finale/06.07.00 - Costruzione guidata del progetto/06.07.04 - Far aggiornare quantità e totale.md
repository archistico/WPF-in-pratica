# 06.07.04 - Far aggiornare quantità e totale

## Il progetto ora deve diventare davvero reattivo

Nel passo precedente abbiamo ottenuto una cassa che permette di aggiungere piatti al tavolo corrente.
È un passaggio importante, ma ancora incompleto.

Adesso vogliamo far percepire al lettore il vero vantaggio del modello WPF:
la schermata deve reagire subito a quantità e totale che cambiano.

Questo è il momento in cui `INotifyPropertyChanged`, proprietà derivate e comandi iniziano a collaborare davvero.

## L'obiettivo di questo passo

Vogliamo ottenere:
- aumento e diminuzione della quantità;
- rimozione della riga se la quantità arriva a zero;
- totale che cambia subito;
- selezione della riga come contesto operativo locale.

## Serve una proprietà di selezione della riga

Nel `ViewModel` aggiungiamo:

```csharp
private RigaConto? _rigaSelezionata;

public RigaConto? RigaSelezionata
{
    get => _rigaSelezionata;
    set
    {
        if (_rigaSelezionata != value)
        {
            _rigaSelezionata = value;
            OnPropertyChanged();
        }
    }
}
```

Questa proprietà è molto importante perché rende chiaro il rapporto tra `ListBox` o `DataGrid` e i comandi di modifica.

## Aggiungiamo i comandi di quantità

```csharp
public ICommand AumentaQuantitaCommand { get; }
public ICommand DiminuisciQuantitaCommand { get; }

public CassaRistoranteViewModel()
{
    // altri comandi...

    AumentaQuantitaCommand = new RelayCommand(
        _ => AumentaQuantita(),
        _ => RigaSelezionata is not null);

    DiminuisciQuantitaCommand = new RelayCommand(
        _ => DiminuisciQuantita(),
        _ => RigaSelezionata is not null);
}
```

## I metodi di modifica

```csharp
private void AumentaQuantita()
{
    if (RigaSelezionata is null)
        return;

    RigaSelezionata.Quantita++;
    OnPropertyChanged(nameof(TotaleCorrente));
}

private void DiminuisciQuantita()
{
    if (TavoloSelezionato is null || RigaSelezionata is null)
        return;

    RigaSelezionata.Quantita--;

    if (RigaSelezionata.Quantita <= 0)
    {
        TavoloSelezionato.RigheConto.Remove(RigaSelezionata);
        RigaSelezionata = null;
    }

    OnPropertyChanged(nameof(RigheContoCorrenti));
    OnPropertyChanged(nameof(TotaleCorrente));
}
```

## Perché la rimozione a quantità zero è una scelta buona

Per il caso finale del libro è una scelta molto sensata perché:
- mantiene il conto pulito;
- evita righe “vuote” senza significato operativo;
- rende il comportamento facile da capire;
- collega bene il comando di decremento a un effetto reale della UI.

In un prodotto più complesso si potrebbero fare scelte diverse, ma qui questa soluzione è ottima.

## La UI ora deve poter selezionare una riga

Anche se finora abbiamo usato una lista semplice, adesso è utile far vedere esplicitamente la selezione:

```xml
<ListBox ItemsSource="{Binding RigheContoCorrenti}"
         SelectedItem="{Binding RigaSelezionata, Mode=TwoWay}">
    <ListBox.ItemTemplate>
        <DataTemplate>
            <Grid Margin="4">
                <Grid.ColumnDefinitions>
                    <ColumnDefinition Width="*"/>
                    <ColumnDefinition Width="80"/>
                    <ColumnDefinition Width="100"/>
                </Grid.ColumnDefinitions>

                <TextBlock Text="{Binding NomePiatto}"/>
                <TextBlock Grid.Column="1"
                           Text="{Binding Quantita}"
                           HorizontalAlignment="Center"/>
                <TextBlock Grid.Column="2"
                           Text="{Binding Subtotale, StringFormat={}{0:C}}"
                           HorizontalAlignment="Right"/>
            </Grid>
        </DataTemplate>
    </ListBox.ItemTemplate>
</ListBox>
```

## I pulsanti di modifica

```xml
<StackPanel Grid.Column="2"
            VerticalAlignment="Bottom">
    <Button Content="Aggiungi piatto"
            Command="{Binding AggiungiPiattoCommand}"
            Height="42"
            Margin="0,0,0,10"/>

    <Button Content="Aumenta quantità"
            Command="{Binding AumentaQuantitaCommand}"
            Height="42"
            Margin="0,0,0,10"/>

    <Button Content="Diminuisci quantità"
            Command="{Binding DiminuisciQuantitaCommand}"
            Height="42"/>
</StackPanel>
```

## Il totale ora diventa davvero significativo

Prima il totale era solo un numero che nasceva quando aggiungevi righe.
Adesso diventa un dato davvero dinamico, perché cambia anche quando:
- aumenti una quantità;
- diminuisci una quantità;
- elimini una riga.

Questo è il momento giusto per far sentire al lettore il valore di una proprietà derivata come:

```csharp
public decimal TotaleCorrente
    => TavoloSelezionato?.RigheConto.Sum(r => r.Subtotale) ?? 0m;
```

## Un punto molto importante sul binding

Qui il lettore vede molto bene una cosa:
la UI non aggiorna il totale perché “qualcuno scrive nel `TextBlock`”.
Lo aggiorna perché il `ViewModel` espone uno stato coerente e notifica i cambiamenti.

Questo è uno dei messaggi più maturi del progetto finale.

## Regola pratica

Quando quantità e totale cominciano ad aggiornarsi in tempo reale, il lettore sente davvero che l’applicazione è viva.
È qui che il progetto smette di essere solo strutturato bene e comincia a sembrare davvero usabile.

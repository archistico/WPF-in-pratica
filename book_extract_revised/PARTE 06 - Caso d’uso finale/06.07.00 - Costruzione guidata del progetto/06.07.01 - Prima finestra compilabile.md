# 06.07.01 - Prima finestra compilabile

## Il punto giusto da cui partire davvero

Se il caso finale vuole somigliare a un progetto vero, non basta mostrarne la forma finale già compiuta.
Bisogna far vedere come nasce.

Il primo passo sensato è quindi costruire una versione minima che:
- compili;
- si apra;
- abbia già una struttura leggibile;
- permetta di intuire la direzione del progetto.

Questa prima versione non deve fare tutto.
Deve però essere abbastanza concreta da far capire che l'applicazione esiste davvero.

## L'obiettivo di questo primo passo

In questa fase vogliamo ottenere:
- una `MainWindow` che si apre correttamente;
- un `DataContext` assegnato;
- una UI ancora semplice ma già organizzata;
- due aree visive distinte: tavoli e conto.

È il primo scheletro funzionante della cassa.

## La struttura minima del progetto

Per questa primissima versione bastano:
- `MainWindow.xaml`
- `MainWindow.xaml.cs`
- `CassaRistoranteViewModel`
- modello `Tavolo`

I modelli `Piatto` e `RigaConto` arriveranno subito dopo, ma per partire possiamo già mostrare l'idea di contesto corrente.

## Il modello minimo `Tavolo`

```csharp
namespace CassaRistorante.Models
{
    public class Tavolo
    {
        public int Id { get; set; }
        public string Nome { get; set; } = "";
    }
}
```

Questa prima versione è volutamente semplice.
Qui non stiamo ancora chiedendo al tavolo di contenere il conto.
Stiamo solo costruendo il primo contesto visibile.

## Il primo `ViewModel`

```csharp
using System.Collections.ObjectModel;
using System.Linq;
using CassaRistorante.Models;

namespace CassaRistorante.ViewModels
{
    public class CassaRistoranteViewModel
    {
        public ObservableCollection<Tavolo> Tavoli { get; } = new();

        public Tavolo? TavoloSelezionato { get; set; }

        public CassaRistoranteViewModel()
        {
            Tavoli.Add(new Tavolo { Id = 1, Nome = "Tavolo 1" });
            Tavoli.Add(new Tavolo { Id = 2, Nome = "Tavolo 2" });
            Tavoli.Add(new Tavolo { Id = 3, Nome = "Tavolo 3" });

            TavoloSelezionato = Tavoli.FirstOrDefault();
        }
    }
}
```

Questo `ViewModel` è ancora povero, ma fa già vedere una cosa importante:
la schermata non nasce dai controlli.
Nasce da uno stato.

## La prima `MainWindow.xaml`

```xml
<Window x:Class="CassaRistorante.MainWindow"
        xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation"
        xmlns:x="http://schemas.microsoft.com/winfx/2006/xaml"
        Title="Cassa Ristorante"
        Height="600"
        Width="1000"
        Background="#FFF5F7FA">

    <Grid Margin="18">
        <Grid.ColumnDefinitions>
            <ColumnDefinition Width="300"/>
            <ColumnDefinition Width="18"/>
            <ColumnDefinition Width="*"/>
        </Grid.ColumnDefinitions>

        <Border Grid.Column="0"
                Background="White"
                BorderBrush="#FFD8DEE8"
                BorderThickness="1"
                CornerRadius="12"
                Padding="16">
            <StackPanel>
                <TextBlock Text="Tavoli"
                           FontSize="22"
                           FontWeight="SemiBold"
                           Margin="0,0,0,12"/>

                <ListBox ItemsSource="{Binding Tavoli}"
                         SelectedItem="{Binding TavoloSelezionato, Mode=TwoWay}"
                         DisplayMemberPath="Nome"/>
            </StackPanel>
        </Border>

        <Border Grid.Column="2"
                Background="White"
                BorderBrush="#FFD8DEE8"
                BorderThickness="1"
                CornerRadius="12"
                Padding="16">
            <StackPanel>
                <TextBlock Text="Conto corrente"
                           FontSize="22"
                           FontWeight="SemiBold"
                           Margin="0,0,0,12"/>

                <TextBlock Text="{Binding TavoloSelezionato.Nome, StringFormat=Tavolo attivo: {0}}"
                           FontSize="16"
                           Foreground="#FF475467"/>

                <TextBlock Margin="0,20,0,0"
                           Text="Qui comparirà il conto del tavolo selezionato."
                           Foreground="#FF667085"/>
            </StackPanel>
        </Border>
    </Grid>
</Window>
```

## Il code-behind minimo

```csharp
using System.Windows;
using CassaRistorante.ViewModels;

namespace CassaRistorante
{
    public partial class MainWindow : Window
    {
        public MainWindow()
        {
            InitializeComponent();
            DataContext = new CassaRistoranteViewModel();
        }
    }
}
```

## Perché questa prima versione è didatticamente giusta

Perché:
- compila subito;
- mostra il contesto;
- usa già `DataContext`;
- usa già `ListBox` e binding;
- introduce una struttura di layout sensata;
- non prova ancora a fare troppo.

Questo è molto importante.
Un progetto finale ben spiegato non comincia con la versione completa.
Comincia con una versione piccola ma credibile.

## Che cosa il lettore deve vedere in questa fase

Il lettore deve vedere che:
- il progetto parte da uno stato semplice;
- la finestra esiste davvero;
- la selezione del tavolo ha già un effetto;
- il layout ha già una logica.

Questo è il primo passo concreto del caso finale.

## Regola pratica

La prima versione di un progetto WPF finale deve essere piccola ma vera:
deve compilare, aprirsi e mostrare già il cuore del contesto operativo, senza voler introdurre tutto subito.

# 06.07.02 - Inserire i tavoli e vedere il contesto cambiare

## Il primo vero effetto della selezione

Nella versione iniziale, la finestra si apriva e mostrava un elenco di tavoli.
Adesso facciamo un passo in più: rendiamo il tavolo un vero contesto attivo, con stato e dati che cambiano quando la selezione cambia.

Questo passaggio è molto importante, perché è il momento in cui la schermata smette di essere solo un layout con una lista e diventa davvero una view contestuale.

## L'obiettivo di questo passo

Vogliamo ottenere tre cose:
- ogni tavolo deve avere uno stato più ricco;
- la UI deve riflettere il tavolo selezionato;
- il lettore deve vedere chiaramente che la selezione governa il resto della schermata.

## Il modello `Tavolo` migliora

Ora aggiungiamo almeno:
- stato occupato/libero;
- totale attuale del tavolo.

Per questa fase possiamo ancora simulare i dati, senza introdurre già tutte le righe del conto.

```csharp
using System.ComponentModel;
using System.Runtime.CompilerServices;

namespace CassaRistorante.Models
{
    public class Tavolo : INotifyPropertyChanged
    {
        private int _id;
        private string _nome = "";
        private bool _isOccupato;
        private decimal _totale;

        public int Id
        {
            get => _id;
            set
            {
                if (_id != value)
                {
                    _id = value;
                    OnPropertyChanged();
                }
            }
        }

        public string Nome
        {
            get => _nome;
            set
            {
                if (_nome != value)
                {
                    _nome = value;
                    OnPropertyChanged();
                }
            }
        }

        public bool IsOccupato
        {
            get => _isOccupato;
            set
            {
                if (_isOccupato != value)
                {
                    _isOccupato = value;
                    OnPropertyChanged();
                    OnPropertyChanged(nameof(StatoDescrittivo));
                }
            }
        }

        public decimal Totale
        {
            get => _totale;
            set
            {
                if (_totale != value)
                {
                    _totale = value;
                    OnPropertyChanged();
                }
            }
        }

        public string StatoDescrittivo => IsOccupato ? "Occupato" : "Libero";

        public event PropertyChangedEventHandler? PropertyChanged;

        protected void OnPropertyChanged([CallerMemberName] string? propertyName = null)
        {
            PropertyChanged?.Invoke(this, new PropertyChangedEventArgs(propertyName));
        }
    }
}
```

## Il `ViewModel` ora inizializza tavoli più realistici

```csharp
using System.Collections.ObjectModel;
using System.ComponentModel;
using System.Linq;
using System.Runtime.CompilerServices;
using CassaRistorante.Models;

namespace CassaRistorante.ViewModels
{
    public class CassaRistoranteViewModel : INotifyPropertyChanged
    {
        private Tavolo? _tavoloSelezionato;

        public ObservableCollection<Tavolo> Tavoli { get; } = new();

        public Tavolo? TavoloSelezionato
        {
            get => _tavoloSelezionato;
            set
            {
                if (_tavoloSelezionato != value)
                {
                    _tavoloSelezionato = value;
                    OnPropertyChanged();
                }
            }
        }

        public CassaRistoranteViewModel()
        {
            Tavoli.Add(new Tavolo { Id = 1, Nome = "Tavolo 1", IsOccupato = false, Totale = 0m });
            Tavoli.Add(new Tavolo { Id = 2, Nome = "Tavolo 2", IsOccupato = true, Totale = 23.50m });
            Tavoli.Add(new Tavolo { Id = 3, Nome = "Tavolo 3", IsOccupato = true, Totale = 41.00m });

            TavoloSelezionato = Tavoli.FirstOrDefault();
        }

        public event PropertyChangedEventHandler? PropertyChanged;

        protected void OnPropertyChanged([CallerMemberName] string? propertyName = null)
        {
            PropertyChanged?.Invoke(this, new PropertyChangedEventArgs(propertyName));
        }
    }
}
```

## La lista tavoli diventa più informativa

Adesso vale la pena migliorare il `ItemTemplate` della `ListBox`, così il contesto è più leggibile a colpo d'occhio.

```xml
<ListBox ItemsSource="{Binding Tavoli}"
         SelectedItem="{Binding TavoloSelezionato, Mode=TwoWay}"
         BorderThickness="0"
         Background="Transparent"
         HorizontalContentAlignment="Stretch">
    <ListBox.ItemTemplate>
        <DataTemplate>
            <Border Margin="0,0,0,10"
                    Padding="14"
                    CornerRadius="10"
                    BorderThickness="1"
                    BorderBrush="#FFE2E8F0"
                    Background="#FFF8FAFC">
                <Grid>
                    <Grid.ColumnDefinitions>
                        <ColumnDefinition Width="*"/>
                        <ColumnDefinition Width="Auto"/>
                    </Grid.ColumnDefinitions>

                    <StackPanel>
                        <TextBlock Text="{Binding Nome}"
                                   FontSize="16"
                                   FontWeight="SemiBold"/>
                        <TextBlock Margin="0,4,0,0"
                                   Text="{Binding Totale, StringFormat=Totale: {0:C}}"
                                   Foreground="#FF667085"/>
                    </StackPanel>

                    <Border Grid.Column="1"
                            Padding="8,4"
                            CornerRadius="8"
                            Background="#FFE8EEF8">
                        <TextBlock Text="{Binding StatoDescrittivo}"
                                   Foreground="#FF1D4ED8"/>
                    </Border>
                </Grid>
            </Border>
        </DataTemplate>
    </ListBox.ItemTemplate>
</ListBox>
```

## Anche la parte destra migliora

Ora il pannello del conto può già mostrare più chiaramente il contesto attivo.

```xml
<StackPanel>
    <TextBlock Text="Conto corrente"
               FontSize="22"
               FontWeight="SemiBold"
               Margin="0,0,0,12"/>

    <TextBlock Text="{Binding TavoloSelezionato.Nome, StringFormat=Tavolo attivo: {0}}"
               FontSize="16"
               Foreground="#FF475467"/>

    <TextBlock Margin="0,6,0,0"
               Text="{Binding TavoloSelezionato.StatoDescrittivo, StringFormat=Stato: {0}}"
               Foreground="#FF667085"/>

    <TextBlock Margin="0,6,0,0"
               Text="{Binding TavoloSelezionato.Totale, StringFormat=Totale corrente: {0:C}}"
               FontWeight="SemiBold"/>
</StackPanel>
```

## Che cosa cambia davvero dal punto di vista didattico

Qui il lettore vede una cosa molto importante:
selezionare una voce in una `ListBox` non è solo un fatto visivo.
Significa cambiare il contesto di tutta la schermata.

Questo è un punto fortissimo in WPF, e va visto bene adesso, prima ancora di introdurre i piatti e le righe conto.

## Perché non stiamo ancora usando il conto vero

Perché è utile che il lettore veda i passi in sequenza.
Prima impariamo che:
- esiste un contesto attivo;
- la selezione cambia la view;
- la UI riflette lo stato corrente.

Subito dopo introdurremo il conto vero come struttura dinamica.

## Regola pratica

Prima di costruire il conto completo, fai sì che il lettore senta davvero che i tavoli sono un contesto di lavoro vivo.
Quando la selezione cambia e la schermata reagisce, il progetto ha già iniziato a comportarsi come una vera applicazione.

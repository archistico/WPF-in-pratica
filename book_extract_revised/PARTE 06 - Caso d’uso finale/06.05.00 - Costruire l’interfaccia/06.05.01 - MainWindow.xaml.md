# 06.05.01 - `MainWindow.xaml`

## La view finale prende forma

Dopo aver costruito:
- i modelli;
- il `RelayCommand`;
- il `ViewModel`;
- le collezioni osservabili;
- proprietà e comandi;

è arrivato il momento di costruire davvero la view principale del caso finale:
`MainWindow.xaml`.

Questo è un passaggio molto importante, perché qui il lettore vede finalmente la schermata prendere forma in modo completo e concreto.

## Obiettivo della view

La view deve permettere di:
- selezionare un tavolo;
- scegliere un piatto;
- vedere il conto corrente;
- modificare quantità;
- rimuovere righe;
- controllare il totale;
- chiudere il conto.

Tutto questo deve avvenire in modo leggibile e ben distribuito nello spazio.

## Una scelta progettuale chiara

Per questa fase finale del libro, conviene usare una `MainWindow.xaml`:
- chiara;
- compatta;
- leggibile;
- già credibile come applicazione reale;
- non eccessivamente frammentata in mille viste.

Questo aiuta molto il lettore a vedere l'insieme senza disperdersi.

## Il punto di partenza: associare il `DataContext`

Nel code-behind della finestra, il `DataContext` viene assegnato al `CassaRistoranteViewModel`.

### `MainWindow.xaml.cs`

```csharp
using System.Windows;
using CassaRistorante.ViewModels;

namespace CassaRistorante
{
    public partial class MainWindow : Window
    {
        public MainWindow()
        {
            InitializeComponent();
            DataContext = new CassaRistoranteViewModel();
        }
    }
}
```

Questo è sufficiente per permettere alla view di bindare le proprietà e i comandi del caso finale.

## Perché questa `MainWindow.xaml` è una buona scelta finale

Perché mostra in un solo file:
- una finestra reale e credibile;
- una gerarchia visiva chiara;
- binding di collezioni e proprietà;
- comandi;
- `DataGrid`;
- aree funzionali ben distinte.

Questo la rende perfetta come caso finale di libro:
è ricca, ma ancora leggibile.

## Una nota importante sull'etichetta di stato del tavolo

Nel template dei tavoli, per una versione completa e corretta potresti voler mostrare “Libero” o “Occupato” con un converter oppure con una proprietà testuale sul modello `Tavolo`.

Per esempio:

```csharp
public string StatoDescrittivo => IsOccupato ? "Occupato" : "Libero";
```

e poi nello XAML:

```xml
<TextBlock Text="{Binding StatoDescrittivo}" />
```

Questa è una piccola rifinitura molto sensata da fare.

## Perché qui non abbiamo ancora estratto stili e risorse

Potremmo già spostare:
- brush;
- stili button;
- bordi;
- font;
- template;

in `ResourceDictionary`.

Ma in questo punto del libro è più utile che il lettore veda tutta la schermata in un file leggibile.
Le rifiniture di riuso visivo arriveranno nel sottocapitolo dedicato.

## Regola pratica

Una buona `MainWindow.xaml` finale deve far vedere l'insieme della schermata in modo chiaro.
Se il lettore riesce a leggere il file e a riconoscere subito:
- contesto;
- azioni;
- conto corrente;
- totale;

allora la view è costruita bene.

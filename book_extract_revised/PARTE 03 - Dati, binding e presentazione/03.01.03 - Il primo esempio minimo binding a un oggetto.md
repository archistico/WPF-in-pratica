# 03.01.03 - Il primo esempio minimo: binding a un oggetto

## L'obiettivo di questo primo esempio

Prima di passare a `DataContext`, binding bidirezionale e aggiornamento automatico della UI, conviene vedere un esempio minimo completo.

L'obiettivo è semplicissimo:
- creare un oggetto;
- assegnarlo come sorgente dati;
- mostrare un suo valore nella UI.

Non ci interessa ancora la complessità. Ci interessa capire bene il meccanismo base.

## La classe del modello

Partiamo da una classe molto semplice:

```csharp
public class Persona
{
    public string Nome { get; set; }
    public string Cognome { get; set; }
}
```

Questa classe rappresenta un piccolo oggetto dati.

## Il code-behind

Nel costruttore della finestra assegniamo un'istanza di `Persona` al `DataContext`.

```csharp
public partial class MainWindow : Window
{
    public MainWindow()
    {
        InitializeComponent();

        DataContext = new Persona
        {
            Nome = "Anna",
            Cognome = "Rossi"
        };
    }
}
```

Per ora non c'è niente di sofisticato.
Stiamo solo dicendo:
“la finestra lavorerà con questo oggetto”.

## Lo XAML

Ora colleghiamo due controlli alle proprietà dell'oggetto:

```xml
<StackPanel Margin="20">
    <TextBlock Text="{Binding Nome}" FontSize="18" />
    <TextBlock Text="{Binding Cognome}" FontSize="18" Margin="0,8,0,0" />
</StackPanel>
```

## Che cosa succede davvero

Quando la finestra viene creata:
1. `InitializeComponent()` costruisce la UI;
2. assegni un oggetto `Persona` al `DataContext`;
3. i binding cercano `Nome` e `Cognome` in quell'oggetto;
4. i valori vengono mostrati nei `TextBlock`.

È tutto qui.

Questa semplicità è molto importante da vedere bene: il binding, nella sua forma base, è più lineare di quanto sembri.

## Perché questo esempio è utile

È utile perché mostra già tre cose fondamentali:
- la UI non contiene i dati in sé;
- i controlli possono leggere proprietà di un oggetto;
- il `DataContext` è il ponte implicito che rende possibile il binding semplice.

## Una variante con `TextBox`

Lo stesso esempio può essere riscritto con controlli editabili:

```xml
<StackPanel Margin="20">
    <TextBox Text="{Binding Nome}" Width="200" Margin="0,0,0,8" />
    <TextBox Text="{Binding Cognome}" Width="200" />
</StackPanel>
```

A questo punto i valori appaiono anche nelle `TextBox`.

Il comportamento però non è ancora completamente “vivo” nel senso che vedremo dopo, perché non abbiamo ancora introdotto `INotifyPropertyChanged`.

## Cosa non stiamo ancora affrontando

Questo primo esempio non copre ancora:
- aggiornamento automatico della UI quando l'oggetto cambia;
- differenza tra `OneWay` e `TwoWay`;
- `UpdateSourceTrigger`;
- binding annidati o più ricchi.

Va bene così.
Qui vogliamo solo fissare bene il primo meccanismo.

## Errore tipico

Un errore comune è voler partire subito con esempi troppo ricchi:
modelli complessi, liste, converter, notifiche, MVVM.

Il rischio è che il lettore non veda più la struttura elementare del binding.

## Regola pratica

Il primo binding che devi saper leggere mentalmente è questo:
- c'è un oggetto;
- l'oggetto viene reso disponibile alla UI;
- i controlli leggono le sue proprietà.

Tutto il resto si costruisce sopra questa base.

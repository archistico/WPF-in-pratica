# 03.05.08 - Grouping

## Quando i dati non vanno solo ordinati, ma organizzati per gruppi

Ordinare mette gli elementi in una sequenza.
Filtrare ne seleziona una parte.
Raggruppare fa un'altra cosa ancora:
organizza gli elementi in insiemi visivi basati su una proprietà comune.

Questo è molto utile quando vuoi rendere la lista più leggibile e strutturata.

## Che cosa significa raggruppare

Raggruppare significa dire alla view:
“metti insieme gli elementi che condividono questo valore”.

Per esempio:
- clienti per città;
- documenti per stato;
- prodotti per categoria;
- movimenti per data;
- attività per progetto.

## Come si applica

Il grouping si applica alla view tramite `GroupDescriptions`.

Esempio:

```csharp
view.GroupDescriptions.Add(new PropertyGroupDescription("Città"));
```

Questo dice alla view di creare gruppi in base alla proprietà `Città`.

## Un esempio concreto

```csharp
public class Cliente
{
    public string Nome { get; set; } = "";
    public string Città { get; set; } = "";
}
```

```csharp
public ObservableCollection<Cliente> Clienti { get; } = new();

public MainWindow()
{
    InitializeComponent();

    Clienti.Add(new Cliente { Nome = "Anna", Città = "Aosta" });
    Clienti.Add(new Cliente { Nome = "Marco", Città = "Torino" });
    Clienti.Add(new Cliente { Nome = "Luca", Città = "Aosta" });
    Clienti.Add(new Cliente { Nome = "Zoe", Città = "Torino" });

    DataContext = this;

    var view = CollectionViewSource.GetDefaultView(Clienti);
    view.GroupDescriptions.Add(new PropertyGroupDescription("Città"));
}
```

## Che cosa succede nella UI

La view non mostrerà più solo una sequenza piatta di elementi.
Mostrerà gruppi:
- Aosta
  - Anna
  - Luca
- Torino
  - Marco
  - Zoe

Naturalmente per visualizzarlo bene serve anche definire come mostrare i gruppi.
Qui entra in gioco `GroupStyle`, che vedremo nel sottocapitolo successivo.

## Perché il grouping è molto utile

È utile quando la lista inizia a essere troppo ricca o lunga per essere letta come semplice sequenza.

Per esempio, in una schermata operativa è molto più leggibile vedere:
- ordini raggruppati per cliente;
- task raggruppati per progetto;
- documenti raggruppati per stato.

## Grouping e ordinamento possono convivere

Molto spesso il grouping funziona ancora meglio se affiancato da sorting.

Per esempio:
- gruppi per città;
- elementi dentro ogni gruppo ordinati per nome.

Questo è un pattern molto comune e molto efficace.

## Quando non serve

Non serve quando:
- la lista è piccola;
- il gruppo non aggiunge chiarezza;
- la proprietà scelta non ha un valore organizzativo reale;
- rischi di complicare una UI che era già leggibile.

## Errori tipici

### Raggruppare per una proprietà poco significativa
La UI diventa più pesante senza diventare più chiara.

### Dimenticare la presentazione dei gruppi
Il grouping c'è, ma visivamente non è chiaro.

### Usare il grouping per nascondere problemi di modello o di layout
Il grouping deve organizzare, non mascherare.

## Regola pratica

Raggruppa quando il gruppo aiuta davvero la lettura dei dati.
Se la UI diventa più comprensibile grazie ai gruppi, allora stai usando bene il modello.

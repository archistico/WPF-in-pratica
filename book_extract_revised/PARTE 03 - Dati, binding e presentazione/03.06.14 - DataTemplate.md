# 03.06.14 - `DataTemplate`

## Non ridefinisce un controllo, ma il modo in cui mostri un dato

Dopo `ControlTemplate`, arriva un altro template fondamentale:
`DataTemplate`.

La differenza è cruciale:
- `ControlTemplate` ridefinisce la struttura visiva di un controllo;
- `DataTemplate` definisce come rappresentare visivamente un oggetto dati.

Questa è una delle distinzioni più importanti di tutta WPF.

## A cosa serve davvero

`DataTemplate` serve quando hai un controllo che mostra dati, per esempio:
- `ListBox`
- `ListView`
- `ItemsControl`
- `ComboBox`
- `ContentControl`

e non vuoi limitarti a una semplice proprietà testuale.

Vuoi invece decidere come visualizzare l'oggetto intero.

## Un esempio semplice

Supponiamo di avere una classe:

```csharp
public class Cliente
{
    public string Nome { get; set; } = "";
    public string Città { get; set; } = "";
}
```

E una lista di clienti.

Con un `DataTemplate` puoi dire come mostrare ogni elemento:

```xml
<ListBox ItemsSource="{Binding Clienti}">
    <ListBox.ItemTemplate>
        <DataTemplate>
            <StackPanel Margin="4">
                <TextBlock Text="{Binding Nome}" FontWeight="SemiBold"/>
                <TextBlock Text="{Binding Città}" FontSize="12"/>
            </StackPanel>
        </DataTemplate>
    </ListBox.ItemTemplate>
</ListBox>
```

## Che cosa sta succedendo

Per ogni oggetto `Cliente` nella collezione:
- il template viene applicato;
- il `DataContext` interno del template è quell'oggetto specifico;
- i binding come `{Binding Nome}` e `{Binding Città}` lavorano sul singolo elemento.

Questo è il punto chiave del `DataTemplate`.

## Perché è così importante

Perché permette di passare da:
- un elenco piatto di stringhe

a:
- una vera rappresentazione visiva ricca degli oggetti dati.

Ed è proprio uno dei punti in cui WPF diventa molto potente.

## Quando usarlo

Usalo quando:
- un semplice `DisplayMemberPath` non basta;
- vuoi mostrare più proprietà insieme;
- vuoi una resa più ricca degli elementi;
- vuoi controllare davvero come viene presentato ogni oggetto.

## Quando può non servire

Se ti basta mostrare una sola proprietà semplice, `DisplayMemberPath` può essere sufficiente.
Ma appena vuoi struttura visiva, `DataTemplate` è quasi sempre la strada giusta.

## Un confronto utile

### `ControlTemplate`
Cambia come appare il controllo.

### `DataTemplate`
Cambia come appare il dato dentro un controllo che mostra dati.

Questa frase va fissata molto bene.

## Errori tipici

### Confondere `DataTemplate` e `ControlTemplate`
È l'errore più classico.

### Dimenticare che il `DataContext` del template è il singolo elemento dati
I binding risultano incomprensibili.

### Usare template pesanti quando basta una presentazione più semplice
Come sempre, serve misura.

## Regola pratica

Usa `DataTemplate` quando vuoi decidere come mostrare un oggetto dati.
Se la domanda è “come visualizzo ogni elemento di questa lista?”, sei quasi certamente nel territorio giusto.

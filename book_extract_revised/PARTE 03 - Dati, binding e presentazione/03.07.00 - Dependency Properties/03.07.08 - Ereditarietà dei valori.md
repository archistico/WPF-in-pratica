# 03.07.08 - Ereditarietà dei valori

## Un comportamento molto tipico del property system di WPF

Uno degli aspetti più interessanti delle dependency property è che alcune possono ereditare il proprio valore lungo l'albero degli elementi.

Questo significa che un elemento figlio può ricevere un valore non perché lo abbia impostato direttamente, ma perché lo eredita da un antenato.

È un comportamento molto “WPF” e molto diverso da una proprietà normale pensata in modo puramente locale.

## L'esempio più famoso: `DataContext`

```xml
<Grid DataContext="{Binding ClienteCorrente}">
    <StackPanel>
        <TextBox Text="{Binding Nome}" />
        <TextBox Text="{Binding Cognome}" />
    </StackPanel>
</Grid>
```

Qui il `TextBox` non ha un `DataContext` impostato localmente.
Eppure riesce a usarlo, perché `DataContext` è una dependency property che può ereditare il valore lungo l'albero.

Questo è uno degli esempi più formativi di tutto il capitolo.

## Perché questo comportamento è così utile

Perché evita di ripetere continuamente lo stesso valore su ogni controllo figlio.

In pratica permette di dire:
“da qui in poi, questi elementi lavorano con questo contesto”,
oppure:
“da qui in poi, certi valori visivi o comportamentali si propagano”.

Questo rende la UI molto più dichiarativa e meno ripetitiva.

## Non tutte le dependency property ereditano

Questo è importante.
Il fatto che una proprietà sia una dependency property non significa automaticamente che erediti il valore.

L'ereditarietà è un comportamento speciale che viene abilitato tramite metadati adeguati.

Quindi:
- alcune dependency property ereditano;
- molte altre no.

## Un altro esempio mentale

Immagina una proprietà visiva o contestuale che vuoi far discendere ai figli senza reimpostarla ovunque.
Il property system di WPF può supportare questo scenario proprio tramite l'ereditarietà del valore.

Questo è uno dei casi in cui le dependency property mostrano bene di essere parte di un sistema più ricco di una proprietà locale.

## Perché una proprietà CLR normale non offre questo modello

Una proprietà normale non ha un meccanismo nativo che dica:
“se qui non è impostata, guarda al padre e prova a ereditare”.

Questo tipo di comportamento richiede un sistema di risoluzione più avanzato, che è proprio ciò che il property system di WPF fornisce.

## Quando questo punto è importante per te

È importante quando:
- studi il comportamento di proprietà come `DataContext`;
- costruisci controlli personalizzati con logiche di propagazione;
- vuoi capire come WPF risolve certi valori senza ripetizione esplicita;
- inizi a leggere metadati avanzati delle dependency property.

## Errori tipici

### Pensare che tutte le dependency property ereditino il valore
Non è vero.

### Non capire che certi binding funzionano grazie a questo meccanismo
Il comportamento di WPF sembra magico, quando invece è sistematico.

### Confondere ereditarietà dei valori con ereditarietà OO
Sono concetti completamente diversi.

## Regola pratica

L'ereditarietà dei valori è una caratteristica speciale di alcune dependency property e mostra molto bene quanto il property system di WPF sia più ricco di un semplice insieme di proprietà CLR.

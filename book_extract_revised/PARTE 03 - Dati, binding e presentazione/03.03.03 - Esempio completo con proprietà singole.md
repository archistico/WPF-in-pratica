# 03.03.03 - Esempio completo con proprietà singole

## Obiettivo dell'esempio

Qui vogliamo vedere un caso completo con:
- proprietà singole;
- `INotifyPropertyChanged`;
- binding reale;
- aggiornamento della UI sia da input utente sia da codice.

Questo è uno degli esempi più importanti del capitolo, perché mostra in modo molto chiaro il meccanismo corretto.

## Il modello

```csharp
using System.ComponentModel;
using System.Runtime.CompilerServices;

public class Cliente : INotifyPropertyChanged
{
    private string _nome = "";
    private string _città = "";

    public string Nome
    {
        get => _nome;
        set
        {
            if (_nome != value)
            {
                _nome = value;
                OnPropertyChanged();
            }
        }
    }

    public string Città
    {
        get => _città;
        set
        {
            if (_città != value)
            {
                _città = value;
                OnPropertyChanged();
            }
        }
    }

    public event PropertyChangedEventHandler? PropertyChanged;

    protected void OnPropertyChanged([CallerMemberName] string? propertyName = null)
    {
        PropertyChanged?.Invoke(this, new PropertyChangedEventArgs(propertyName));
    }
}
```

## Il code-behind

```csharp
public partial class MainWindow : Window
{
    private Cliente _cliente;

    public MainWindow()
    {
        InitializeComponent();

        _cliente = new Cliente
        {
            Nome = "Anna",
            Città = "Torino"
        };

        DataContext = _cliente;
    }

    private void CambiaDati_Click(object sender, RoutedEventArgs e)
    {
        _cliente.Nome = "Luca";
        _cliente.Città = "Aosta";
    }
}
```

## Lo XAML

```xml
<StackPanel Margin="20">
    <TextBlock Text="Nome" Margin="0,0,0,4"/>
    <TextBox Text="{Binding Nome, Mode=TwoWay}"
             Width="220"
             Margin="0,0,0,10"/>

    <TextBlock Text="Città" Margin="0,0,0,4"/>
    <TextBox Text="{Binding Città, Mode=TwoWay}"
             Width="220"
             Margin="0,0,0,10"/>

    <TextBlock Text="{Binding Nome, StringFormat=Cliente: {0}}"
               FontWeight="SemiBold"
               Margin="0,8,0,4"/>

    <TextBlock Text="{Binding Città, StringFormat=Città: {0}}"
               Margin="0,0,0,12"/>

    <Button Content="Cambia da codice"
            Width="150"
            Click="CambiaDati_Click"/>
</StackPanel>
```

## Che cosa dimostra questo esempio

### 1. Il dato iniziale compare nella UI
All'avvio vedi `"Anna"` e `"Torino"`.

### 2. Le `TextBox` modificano il modello
Grazie al binding `TwoWay`, ciò che scrive l'utente viene riportato nelle proprietà.

### 3. Il modello aggiorna la UI
Quando premi il pulsante, il code-behind cambia le proprietà del modello e la UI si aggiorna automaticamente.

Questo è il punto decisivo:
la UI non viene aggiornata manualmente, ma grazie alla notifica delle proprietà.

## Perché è un esempio corretto da tenere a mente

Perché mostra il comportamento giusto nella forma più leggibile possibile:
- poche proprietà;
- nessuna complessità inutile;
- due direzioni del binding ben visibili;
- `INotifyPropertyChanged` usato esattamente dove serve.

## Quando usare un modello così

Questo approccio è perfetto per:
- schede dati;
- finestre di modifica;
- pannelli impostazioni;
- maschere anagrafiche;
- UI che devono restare vive mentre il modello cambia.

## Errori tipici

### Togliere `INotifyPropertyChanged`
La UI smette di aggiornarsi quando il dato cambia da codice.

### Lasciare auto-property semplici
Il modello non può notificare i cambiamenti.

### Pensare che il pulsante stia aggiornando la UI direttamente
In realtà aggiorna l'oggetto, ed è il binding a fare il resto.

## Regola pratica

Se vuoi una UI che segua davvero il cambiamento di proprietà singole, questo è il modello base che devi saper costruire senza esitazioni.

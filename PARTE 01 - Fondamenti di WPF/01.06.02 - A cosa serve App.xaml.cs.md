# 01.06.02 - A cosa serve `App.xaml.cs`

Se `App.xaml` è la parte dichiarativa del livello applicazione, `App.xaml.cs` è la sua parte in C#. È il **code-behind** della classe `App`.

All'inizio questo file può sembrare quasi inutile, perché spesso Visual Studio lo crea con pochissimo codice:

```csharp
using System.Windows;

namespace EsempioWpf
{
    public partial class App : Application
    {
    }
}
```

Eppure quel file ha un ruolo preciso. Rappresenta la classe applicazione e diventa importante quando il comportamento dell'app non può essere descritto bene solo in XAML.

Per esempio, `App.xaml.cs` è il posto giusto quando vuoi:

- eseguire inizializzazioni globali;
- scegliere a runtime quale finestra aprire;
- leggere argomenti da riga di comando;
- intercettare eventi di avvio e chiusura;
- gestire eccezioni non intercettate del thread UI;
- impostare il comportamento di shutdown.

In altre parole, qui non descrivi **com'è fatta una finestra**, ma **come nasce e vive l'applicazione**.

Esempio:

```csharp
using System.Windows;

namespace EsempioWpf
{
    public partial class App : Application
    {
        protected override void OnStartup(StartupEventArgs e)
        {
            base.OnStartup(e);

            var finestra = new MainWindow();
            MainWindow = finestra;
            finestra.Show();
        }
    }
}
```

Questo codice non costruisce il layout della finestra: decide invece la strategia di avvio dell'app.

Un errore tipico è spostare in `App.xaml.cs` logiche che appartengono alla finestra o alla UI di dettaglio. Per esempio, validazioni di una `TextBox`, gestione di pulsanti o codice fortemente legato a una singola schermata non dovrebbero stare qui.

La distinzione da ricordare è netta:

- `App.xaml.cs` = livello applicazione;
- `MainWindow.xaml.cs` = livello finestra.

Capire presto questa separazione aiuta a tenere il progetto più pulito e più leggibile.

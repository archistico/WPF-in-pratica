# 01.04.13 - Errori di sintassi molto comuni in XAML

XAML è leggibile, ma richiede precisione. Anche errori piccoli possono bloccare compilazione, designer o caricamento dell’interfaccia.

## Tag non chiusi correttamente

Uno degli errori più comuni è dimenticare la chiusura corretta di un elemento.

Sbagliato:

```xml
<StackPanel>
    <TextBox Width="200">
</StackPanel>
```

Corretto:

```xml
<StackPanel>
    <TextBox Width="200" />
</StackPanel>
```

## Annidamento errato

Se apri un elemento dentro un altro, devi chiuderli nell’ordine giusto.

Sbagliato:

```xml
<Grid>
    <StackPanel>
</Grid>
    </StackPanel>
```

## Attributi scritti male

Un nome di proprietà errato o un valore non valido possono causare errori immediati.

Per esempio:

```xml
<Button Widht="120" />
```

`Widht` è sbagliato: la proprietà corretta è `Width`.

## Uso improprio di testo e contenuto interno

A volte si inserisce del contenuto interno in controlli che non lo supportano nel modo immaginato. In questi casi il problema non è solo “di forma”, ma anche di modello.

## Event handler mancanti

Se in XAML scrivi:

```xml
<Button Click="ButtonSalva_Click" />
```

ma quel metodo non esiste più nel code-behind, il progetto segnalerà errore.

## Markup extension scritte male

Una parentesi graffa mancante o un nome sbagliato dentro una markup extension manda facilmente in errore il parser.

## Un consiglio pratico importante

Quando XAML dà errore, non leggere il file in modo frettoloso. Controlla con ordine:

1. chiusura dei tag;
2. annidamento;
3. nomi delle proprietà;
4. valori scritti negli attributi;
5. riferimenti a eventi;
6. parentesi graffe e sintassi speciale.

## Cosa portare a casa

Gli errori di sintassi in XAML spesso sono piccoli ma bloccanti. La strategia migliore non è andare nel panico: è rileggere la struttura con metodo.

# 01.06.06 - Gli eventi importanti del livello applicazione

La classe `Application` non serve solo a contenere risorse o a decidere la finestra iniziale. Gestisce anche il **ciclo di vita** dell'applicazione.

Per questo espone alcuni eventi molto utili. All'inizio non serve conoscerli tutti, ma vale la pena fissare quelli più importanti.

## `Startup`

Scatta quando l'applicazione parte.

È il punto giusto per:

- inizializzare servizi;
- leggere argomenti da riga di comando;
- scegliere la prima finestra;
- eseguire controlli preliminari.

Esempio:

```csharp
private void Application_Startup(object sender, StartupEventArgs e)
{
    var finestra = new MainWindow();
    MainWindow = finestra;
    finestra.Show();
}
```

## `Exit`

Scatta quando l'applicazione sta terminando.

Può essere utile per:

- salvare impostazioni;
- scrivere log finali;
- rilasciare risorse.

Esempio:

```csharp
private void Application_Exit(object sender, ExitEventArgs e)
{
    // Salvataggi finali o cleanup
}
```

## `DispatcherUnhandledException`

È uno degli eventi più interessanti in una vera applicazione desktop. Permette di intercettare eccezioni non gestite nel thread UI.

Esempio:

```csharp
private void Application_DispatcherUnhandledException(
    object sender,
    System.Windows.Threading.DispatcherUnhandledExceptionEventArgs e)
{
    MessageBox.Show("Si è verificato un errore imprevisto.");
    e.Handled = true;
}
```

Non sostituisce una buona gestione degli errori, ma può evitare chiusure brutali e aiutare a centralizzare log e messaggi d'errore.

## Altri eventi

Esistono anche eventi come `Activated` e `Deactivated`, che riguardano l'attivazione o la perdita del focus a livello applicazione. All'inizio puoi semplicemente sapere che esistono, senza approfondirli troppo.

L'idea importante da portare a casa è questa: ci sono eventi che appartengono alla singola finestra e altri che appartengono all'intera applicazione. Quelli di `Application` vanno usati quando il comportamento da intercettare è davvero globale.

Errore tipico: usare eventi di livello applicazione per risolvere problemi che in realtà appartengono a una singola `Window` o a un singolo controllo.

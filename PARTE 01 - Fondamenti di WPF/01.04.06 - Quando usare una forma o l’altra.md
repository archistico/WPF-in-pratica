# 01.04.06 - Quando usare una forma o l’altra

Sapere che una proprietà può essere scritta in più modi è utile. Ancora più utile è capire quando conviene scegliere una forma invece dell’altra.

## Regola pratica principale

Usa gli attributi quando il valore è:

- breve;
- immediatamente leggibile;
- privo di struttura interna.

Esempio:

```xml
<TextBlock Text="Benvenuto" FontSize="24" Margin="10" />
```

Usa invece la forma come elemento di proprietà quando il valore è:

- un oggetto complesso;
- una raccolta;
- qualcosa che diventerebbe poco leggibile tutto su una riga.

Esempio:

```xml
<Border>
    <Border.Background>
        <LinearGradientBrush>
            <GradientStop Color="White" Offset="0" />
            <GradientStop Color="LightGray" Offset="1" />
        </LinearGradientBrush>
    </Border.Background>
</Border>
```

## Una questione di leggibilità

A volte entrambe le forme sono tecnicamente valide, ma una delle due è molto più chiara. In un libro e in un progetto reale conviene quasi sempre scegliere la leggibilità.

Per esempio, questa forma è valida ma spesso meno chiara:

```xml
<Button>
    <Button.Margin>10</Button.Margin>
</Button>
```

Molto meglio:

```xml
<Button Margin="10" />
```

## Quando non esagerare con la sintassi compatta

La forma breve è utile, ma non deve diventare una gara a comprimere tutto in una riga. Quando il markup comincia a diventare denso, espandere alcune parti migliora la comprensione.

## Cosa portare a casa

Non scegliere una sintassi solo perché “funziona”. Sceglila perché rende il file più leggibile. In XAML la chiarezza paga sempre.

## 01.02.07 - Che ruolo hanno questi file

Dopo aver visto che Visual Studio crea una struttura iniziale, bisogna chiarire il ruolo dei file principali. Questo passaggio è importante perché aiuta il lettore a non percepire il progetto come un insieme confuso di elementi generati dall’IDE.

## `App.xaml`

È il punto in cui l’applicazione definisce aspetti globali, come risorse condivise e startup. Non descrive una singola finestra: descrive l’app nel suo insieme.

## `App.xaml.cs`

È il code-behind associato ad `App.xaml`. Qui possono vivere logiche applicative iniziali, gestione di eventi globali o comportamento legato all’avvio.

## `MainWindow.xaml`

È la definizione visuale della finestra principale. Qui descrivi controlli, layout, proprietà e struttura dell’interfaccia iniziale.

## `MainWindow.xaml.cs`

È il code-behind della finestra principale. Qui finiscono gli handler degli eventi, il comportamento della finestra e, nei casi più semplici, parte della logica collegata alla UI.

## Il file di progetto

Il file di progetto definisce framework di destinazione, riferimenti, pacchetti, proprietà di build e altre impostazioni necessarie alla compilazione. È meno “visibile” rispetto ai file XAML, ma è uno degli elementi più importanti del progetto.

## I file generati e le cartelle tecniche

Durante build ed esecuzione, Visual Studio e il sistema di compilazione producono file temporanei e cartelle tecniche. All’inizio non serve approfondirli troppo, ma è utile sapere che esistono e che fanno parte del ciclo di compilazione.

## Il punto essenziale

Il lettore deve cogliere questa divisione:

- un livello per la struttura della UI;
- un livello per il comportamento;
- un livello per la configurazione del progetto;
- un livello per l’infrastruttura di compilazione.

### Errore tipico

Confondere il ruolo di `App.xaml` e `MainWindow.xaml`, oppure pensare che ogni file XAML serva sempre solo a “disegnare una schermata”. In WPF ci sono file che rappresentano una finestra e file che rappresentano il livello applicativo.

### Regola pratica

Quando apri un progetto WPF, prima di scrivere codice chiediti sempre: questo cambiamento riguarda la finestra, l’applicazione o il progetto?

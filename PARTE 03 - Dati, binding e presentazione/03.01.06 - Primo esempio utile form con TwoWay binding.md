# 03.01.06 - Primo esempio utile: form con `TwoWay` binding

## Qui il binding comincia a mostrare il suo valore pratico

Dopo gli esempi minimi, è il momento di vedere un caso davvero utile:
un piccolo form in cui la UI modifica direttamente un oggetto tramite binding bidirezionale.

Questo è uno dei primi momenti in cui il lettore percepisce davvero il vantaggio del data binding.

## La classe dati

Partiamo da una classe semplice:

```csharp
public class Cliente
{
    public string Nome { get; set; }
    public string Cognome { get; set; }
    public string Città { get; set; }
}
```

## Il code-behind

```csharp
public partial class MainWindow : Window
{
    public MainWindow()
    {
        InitializeComponent();

        DataContext = new Cliente
        {
            Nome = "Anna",
            Cognome = "Rossi",
            Città = "Torino"
        };
    }
}
```

## Lo XAML del form

```xml
<Grid Margin="20">
    <Grid.RowDefinitions>
        <RowDefinition Height="Auto"/>
        <RowDefinition Height="Auto"/>
        <RowDefinition Height="Auto"/>
        <RowDefinition Height="Auto"/>
    </Grid.RowDefinitions>

    <Grid.ColumnDefinitions>
        <ColumnDefinition Width="Auto"/>
        <ColumnDefinition Width="220"/>
    </Grid.ColumnDefinitions>

    <TextBlock Text="Nome" Margin="0,0,12,8" VerticalAlignment="Center"/>
    <TextBox Grid.Column="1"
             Text="{Binding Nome, Mode=TwoWay}"
             Margin="0,0,0,8"/>

    <TextBlock Grid.Row="1"
               Text="Cognome"
               Margin="0,0,12,8"
               VerticalAlignment="Center"/>
    <TextBox Grid.Row="1"
             Grid.Column="1"
             Text="{Binding Cognome, Mode=TwoWay}"
             Margin="0,0,0,8"/>

    <TextBlock Grid.Row="2"
               Text="Città"
               Margin="0,0,12,8"
               VerticalAlignment="Center"/>
    <TextBox Grid.Row="2"
             Grid.Column="1"
             Text="{Binding Città, Mode=TwoWay}"
             Margin="0,0,0,8"/>

    <TextBlock Grid.Row="3"
               Grid.ColumnSpan="2"
               Margin="0,12,0,0"
               FontWeight="SemiBold"
               Text="{Binding Nome}"/>
</Grid>
```

## Che cosa mostra davvero questo esempio

Questo esempio è utile per diversi motivi:

### 1. I controlli leggono i dati iniziali
Le `TextBox` partono con i valori dell'oggetto `Cliente`.

### 2. L'utente modifica direttamente l'oggetto tramite la UI
Il binding `TwoWay` permette di riportare nella source ciò che l'utente inserisce.

### 3. Lo stesso dato può essere usato in più punti
La proprietà `Nome` è usata sia in una `TextBox` sia in un `TextBlock`.

Questo rende visibile l'idea che la UI non è fatta di valori isolati, ma di rappresentazioni multiple dello stesso dato.

## Perché questo è un passo importante

Qui si comincia a vedere la differenza con un approccio totalmente manuale.

Senza binding avresti dovuto:
- caricare i valori nei controlli;
- recuperarli dai controlli;
- riscriverli nell'oggetto;
- aggiornare manualmente altre parti della UI.

Con il binding, il collegamento è dichiarato nel markup.

## Ma c'è un limite importante

Se in questo momento il modello cambia da codice **dopo** il caricamento della finestra, la UI non si aggiornerà automaticamente come ti aspetteresti, almeno non in tutti i casi.

Perché?

Perché manca ancora `INotifyPropertyChanged`.

Ed è proprio questo il passaggio naturale del sottocapitolo successivo.

## Quando usare questo approccio

Questo tipo di binding è perfetto per:
- form anagrafici;
- schermate di modifica;
- finestre impostazioni;
- schede dati;
- interfacce dove il controllo deve leggere e modificare proprietà semplici.

## Errori tipici

### Pensare che `TwoWay` da solo risolva tutto
Per il flusso UI → oggetto sì, ma per l'aggiornamento automatico oggetto → UI in tempo reale serve altro.

### Dimenticare il ruolo del `DataContext`
Il form sembra giusto, ma il binding punta all'oggetto sbagliato.

### Confondere il successo iniziale del binding con un sistema completo di sincronizzazione
Il modello è ancora incompleto finché non introduci le notifiche di cambiamento.

## Regola pratica

Un form con `TwoWay` binding è uno dei primi esempi in cui il vantaggio di WPF diventa concreto.
Ma per farlo vivere davvero bene serve anche il meccanismo che notifichi i cambiamenti alla UI.

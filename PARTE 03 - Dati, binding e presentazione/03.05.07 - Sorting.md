# 03.05.07 - Sorting

## Ordinare è una decisione di presentazione molto tipica

L'ordinamento è uno dei casi più comuni in cui una view è più adatta della modifica diretta dei dati.

Molto spesso non vuoi dire:
“questi dati adesso sono davvero ordinati così”.

Vuoi dire:
“in questa schermata voglio vederli in quest'ordine”.

È una differenza piccola in apparenza, ma molto importante nella progettazione di una UI.

## Come si applica l'ordinamento

Su una view, l'ordinamento si gestisce tramite `SortDescriptions`.

Esempio:

```csharp
view.SortDescriptions.Add(new SortDescription("Nome", ListSortDirection.Ascending));
```

Questo dice alla view di ordinare in base alla proprietà `Nome` in ordine crescente.

## Un esempio completo

```csharp
public class Cliente
{
    public string Nome { get; set; } = "";
    public string Città { get; set; } = "";
}
```

```csharp
public ObservableCollection<Cliente> Clienti { get; } = new();

public MainWindow()
{
    InitializeComponent();

    Clienti.Add(new Cliente { Nome = "Marco", Città = "Torino" });
    Clienti.Add(new Cliente { Nome = "Anna", Città = "Aosta" });
    Clienti.Add(new Cliente { Nome = "Luca", Città = "Milano" });

    DataContext = this;

    var view = CollectionViewSource.GetDefaultView(Clienti);
    view.SortDescriptions.Add(new SortDescription("Nome", ListSortDirection.Ascending));
}
```

XAML:

```xml
<ListView ItemsSource="{Binding Clienti}"
          Margin="20"
          Height="180">
    <ListView.View>
        <GridView>
            <GridViewColumn Header="Nome" DisplayMemberBinding="{Binding Nome}" />
            <GridViewColumn Header="Città" DisplayMemberBinding="{Binding Città}" />
        </GridView>
    </ListView.View>
</ListView>
```

## Più criteri di ordinamento

Una view può usare più criteri.

Per esempio:

```csharp
view.SortDescriptions.Add(new SortDescription("Città", ListSortDirection.Ascending));
view.SortDescriptions.Add(new SortDescription("Nome", ListSortDirection.Ascending));
```

Questo significa:
- prima ordina per città;
- a parità di città, ordina per nome.

È una possibilità molto utile in schermate reali.

## Cambiare ordinamento

Puoi anche cambiare criterio svuotando e ricostruendo i `SortDescriptions`:

```csharp
view.SortDescriptions.Clear();
view.SortDescriptions.Add(new SortDescription("Città", ListSortDirection.Descending));
```

Questa tecnica è molto utile quando l'utente seleziona il criterio da una combo o clicca su un pulsante di ordinamento.

## Perché è meglio di ordinare la collection ogni volta

Perché:
- non alteri l'ordine reale dei dati;
- puoi cambiare facilmente criterio;
- la logica resta nel livello di presentazione;
- il comportamento è coerente con le view di WPF.

## Errori tipici

### Ordinare riscrivendo sempre la collezione
Spesso è più pesante e meno chiaro.

### Dimenticare che l'ordinamento è cumulativo se non svuoti i `SortDescriptions`
Il risultato può sorprendere.

### Scrivere il nome proprietà sbagliato
L'ordinamento non lavora come previsto.

## Regola pratica

Se devi cambiare l'ordine con cui una lista viene mostrata, il primo posto in cui pensarci è quasi sempre la view, non la collezione originale.

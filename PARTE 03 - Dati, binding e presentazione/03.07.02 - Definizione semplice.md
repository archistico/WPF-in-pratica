# 03.07.02 - Definizione semplice

## La definizione più utile da portarsi dietro

Una dependency property è una proprietà gestita dal property system di WPF, non semplicemente da un campo privato e da un getter/setter CLR.

Questa è la definizione tecnica più importante.

Se però vuoi una definizione ancora più pratica, puoi pensare così:

**una dependency property è una proprietà speciale che WPF può usare in modo profondo per binding, stili, animazioni, template e altre funzionalità del framework.**

Questa è spesso la definizione più utile da ricordare all'inizio.

## Perché si chiama “dependency”

Storicamente il nome richiama l'idea che il valore della proprietà possa dipendere da molte fonti diverse.

Per esempio:
- un valore locale;
- uno stile;
- un tema;
- un binding;
- un'animazione;
- l'ereditarietà da un elemento padre.

Quindi il valore non è semplicemente “quello che hai scritto nel setter”.
Può dipendere da un sistema di regole più articolato.

## Un modo corretto di visualizzarla

Non pensare alla dependency property come a una normale proprietà con un nome strano.
Pensala piuttosto come a una proprietà che:
- ha un'identità nel framework;
- è registrata nel property system;
- viene letta e scritta tramite quel sistema;
- può partecipare a molte funzionalità avanzate di WPF.

## Un esempio noto che usi già

Quando scrivi:

```xml
<Button Content="Salva" Width="120" />
```

la proprietà `Width` del `Button` è una dependency property.

Quando scrivi:

```xml
<TextBox Text="{Binding Nome}" />
```

la proprietà `Text` del `TextBox` è una dependency property.

Quindi il punto importante è anche questo:
le dependency property non sono una stranezza rara.
Le stai già usando continuamente.

## Cosa non è

Una dependency property non è:
- semplicemente una proprietà con `INotifyPropertyChanged`;
- una proprietà solo “più osservabile”;
- un meccanismo generico per ogni classe C#.

È uno strumento del sistema UI di WPF, legato agli oggetti che partecipano a quel sistema.

## Perché la definizione semplice è importante

Perché se parti subito dalla sintassi di `Register`, rischi di perdere il senso generale.
Meglio prima capire il ruolo:
la dependency property è il modo in cui WPF tratta certe proprietà come cittadini di prima classe del framework.

## Un piccolo schema mentale

### Proprietà normale
Gestita da C#.

### Dependency property
Gestita da C# **e** dal property system di WPF.

Questo schema è semplificato, ma molto utile.

## Errore tipico

L'errore più comune è memorizzare la definizione solo in termini di sintassi e non di funzione.
La sintassi si dimentica facilmente.
Il ruolo, invece, se è chiaro, guida tutto il resto.

## Regola pratica

Una dependency property è una proprietà che non appartiene solo all'oggetto, ma al sistema di proprietà di WPF.
È questo che la rende speciale.

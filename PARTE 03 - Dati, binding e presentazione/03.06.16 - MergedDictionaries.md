# 03.06.16 - `MergedDictionaries`

## Come comporre più dizionari di risorse in un sistema unico

Dopo aver separato le risorse in file diversi tramite `ResourceDictionary`, nasce una domanda naturale:
come faccio a usare insieme più dizionari?

La risposta è:
`MergedDictionaries`.

Questo meccanismo permette di unire più file di risorse dentro un dizionario principale.

## Perché è così utile

Perché in un progetto reale le risorse vengono spesso suddivise per responsabilità:
- colori;
- brush;
- stili;
- template;
- converter.

`MergedDictionaries` consente di tenere questa separazione e, allo stesso tempo, usare tutto come un sistema coerente.

## Esempio tipico in `App.xaml`

```xml
<Application.Resources>
    <ResourceDictionary>
        <ResourceDictionary.MergedDictionaries>
            <ResourceDictionary Source="Resources/Brushes.xaml"/>
            <ResourceDictionary Source="Resources/Styles.xaml"/>
            <ResourceDictionary Source="Resources/Templates.xaml"/>
        </ResourceDictionary.MergedDictionaries>
    </ResourceDictionary>
</Application.Resources>
```

## Come leggere questo esempio

### Dizionario principale
`Application.Resources` contiene un `ResourceDictionary`.

### Dizionari uniti
Dentro `MergedDictionaries`, elenchi i file da includere.

### Risultato
Le risorse contenute in quei file entrano nel sistema di risorse visibile a livello applicativo.

## Perché è meglio che copiare tutto in un solo file

Perché:
- mantieni separazione tematica;
- riduci il caos in `App.xaml`;
- puoi riordinare meglio il progetto;
- puoi riusare o sostituire gruppi di risorse più facilmente.

## Un caso molto tipico

Potresti avere:
- `Colors.xaml`
- `Brushes.xaml`
- `ButtonStyles.xaml`
- `TextBoxStyles.xaml`
- `Templates.xaml`
- `Converters.xaml`

e poi unirli in `App.xaml`.

Questo è un modo molto sano di organizzare un progetto WPF reale.

## Attenzione all'ordine

L'ordine dei dizionari può contare, soprattutto quando:
- più risorse hanno lo stesso nome;
- alcuni stili dipendono da altre risorse già definite;
- un dizionario usa risorse contenute in un altro.

Per questo conviene pensare anche all'ordine come parte della progettazione.

## Quando usarlo

Usalo quando:
- hai più file di risorse;
- vuoi un sistema centrale ben organizzato;
- l'applicazione ha ormai una struttura UI seria;
- vuoi scalare la manutenzione in modo pulito.

## Quando non serve complicare troppo

Se il progetto è ancora piccolissimo, puoi anche non spezzare subito tutto.
Ma sapere come funziona `MergedDictionaries` è fondamentale per non ritrovarti bloccato più avanti.

## Errori tipici

### Ignorare l'ordine dei dizionari
Può creare problemi di risorse non trovate o override inattesi.

### Spezzare le risorse senza poi ricomporle bene
La struttura resta dispersa.

### Usare `MergedDictionaries` senza una logica di organizzazione
Tanti file, ma nessun vero ordine.

## Regola pratica

`MergedDictionaries` è lo strumento con cui trasformi tanti file di risorse separati in un sistema coerente e manutenibile.
Se le risorse crescono, è quasi inevitabile passare di lì.

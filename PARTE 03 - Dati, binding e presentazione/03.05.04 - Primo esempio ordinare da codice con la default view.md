# 03.05.04 - Primo esempio: ordinare da codice con la default view

## Obiettivo dell'esempio

Qui vogliamo vedere il primo uso concreto della default view:
prendere una collezione già bindata e applicare un ordinamento da codice, senza toccare l'ordine reale dei dati.

Questo è uno degli esempi più utili per far capire davvero la differenza tra collezione e view.

## Il modello

```csharp
public class Cliente
{
    public string Nome { get; set; } = "";
    public string Città { get; set; } = "";
}
```

## La finestra

```csharp
using System.Collections.ObjectModel;
using System.ComponentModel;
using System.Windows.Data;

public partial class MainWindow : Window
{
    public ObservableCollection<Cliente> Clienti { get; } = new();

    public MainWindow()
    {
        InitializeComponent();

        Clienti.Add(new Cliente { Nome = "Marco", Città = "Torino" });
        Clienti.Add(new Cliente { Nome = "Anna", Città = "Aosta" });
        Clienti.Add(new Cliente { Nome = "Luca", Città = "Milano" });

        DataContext = this;

        var view = CollectionViewSource.GetDefaultView(Clienti);
        view.SortDescriptions.Add(new SortDescription("Nome", ListSortDirection.Ascending));
    }
}
```

## Lo XAML

```xml
<ListBox ItemsSource="{Binding Clienti}"
         DisplayMemberPath="Nome"
         Width="220"
         Height="180"
         Margin="20"/>
```

## Che cosa succede davvero

La collezione `Clienti` viene popolata nell'ordine:
- Marco
- Anna
- Luca

Ma la `ListBox`, grazie alla default view ordinata, la mostrerà come:
- Anna
- Luca
- Marco

Questo è il punto cruciale:
hai cambiato la presentazione, non i dati originali.

## Perché è un esempio così importante

Perché mostra in modo molto concreto che:
- il controllo continua a bindarsi alla collezione originale;
- però la presentazione passa attraverso la view;
- ordinare la view non significa riscrivere la collezione.

Questa è una delle idee più eleganti di WPF sulla presentazione dei dati.

## Cosa fa `SortDescriptions`

La collection `SortDescriptions` contiene le regole di ordinamento della view.

In questo caso:

```csharp
view.SortDescriptions.Add(new SortDescription("Nome", ListSortDirection.Ascending));
```

significa:
“ordina in modo crescente in base alla proprietà `Nome`”.

Puoi aggiungere anche più criteri, se necessario.

## Quando questa tecnica è molto utile

È molto utile quando:
- vuoi ordinare una lista mostrata in UI;
- vuoi farlo da codice;
- non vuoi alterare i dati veri;
- vuoi cambiare criterio di ordinamento in modo flessibile.

## Un piccolo passo successivo naturale

Potresti, per esempio, cambiare l'ordinamento in base a un pulsante:
- per nome;
- per città;
- decrescente;
- crescente.

E tutto questo restando sul livello della view.

## Errori tipici

### Aspettarsi che la collezione originale cambi ordine
Non è quello che fa la view.

### Scrivere il nome della proprietà sbagliato in `SortDescription`
L'ordinamento non funzionerà come previsto.

### Manipolare la collezione per ottenere un risultato che la view poteva dare da sola
Si perde chiarezza.

## Regola pratica

Se devi cambiare l'ordine di visualizzazione di una lista, la default view è spesso il primo strumento da considerare.

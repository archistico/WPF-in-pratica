# 03.04.03 - Esempio: `bool` → `Visibility`

## Uno dei converter più classici di WPF

Se c'è un esempio che spiega subito a cosa servono i converter, è questo:
partire da un `bool` e ottenere una `Visibility`.

Il motivo è semplice:
nel modello dati o nel view model è naturale esporre una proprietà come:

```csharp
public bool MostraMessaggio { get; set; }
```

Ma molti controlli WPF non vogliono un `bool` per essere mostrati o nascosti.
Vogliono una `Visibility`, cioè un valore come:
- `Visible`
- `Collapsed`
- `Hidden`

Qui il converter è una soluzione perfetta.

## Il converter

```csharp
using System;
using System.Globalization;
using System.Windows;
using System.Windows.Data;

public class BoolToVisibilityConverter : IValueConverter
{
    public object Convert(object value, Type targetType, object parameter, CultureInfo culture)
    {
        if (value is bool flag && flag)
            return Visibility.Visible;

        return Visibility.Collapsed;
    }

    public object ConvertBack(object value, Type targetType, object parameter, CultureInfo culture)
    {
        if (value is Visibility visibility)
            return visibility == Visibility.Visible;

        return false;
    }
}
```

## Come funziona

### `Convert`
Se il valore è `true`, ritorna `Visibility.Visible`.
Altrimenti ritorna `Visibility.Collapsed`.

### `ConvertBack`
Se arriva una `Visibility`, la riconverte in `bool`.

Questa seconda parte non sempre è necessaria in scenari pratici, ma è utile per capire la struttura completa.

## Registrazione nello XAML

```xml
<Window.Resources>
    <local:BoolToVisibilityConverter x:Key="BoolToVisibilityConverter" />
</Window.Resources>
```

## Uso nel binding

```xml
<TextBlock Text="Attenzione: dati incompleti"
           Visibility="{Binding MostraMessaggio,
                                Converter={StaticResource BoolToVisibilityConverter}}" />
```

## Un piccolo esempio completo

View model o modello:

```csharp
using System.ComponentModel;
using System.Runtime.CompilerServices;

public class SchermataViewModel : INotifyPropertyChanged
{
    private bool _mostraMessaggio;

    public bool MostraMessaggio
    {
        get => _mostraMessaggio;
        set
        {
            if (_mostraMessaggio != value)
            {
                _mostraMessaggio = value;
                OnPropertyChanged();
            }
        }
    }

    public event PropertyChangedEventHandler? PropertyChanged;

    protected void OnPropertyChanged([CallerMemberName] string? propertyName = null)
    {
        PropertyChanged?.Invoke(this, new PropertyChangedEventArgs(propertyName));
    }
}
```

XAML:

```xml
<StackPanel Margin="20">
    <CheckBox Content="Mostra messaggio"
              IsChecked="{Binding MostraMessaggio, Mode=TwoWay}" />

    <TextBlock Margin="0,12,0,0"
               Text="Messaggio importante"
               Visibility="{Binding MostraMessaggio,
                                    Converter={StaticResource BoolToVisibilityConverter}}" />
</StackPanel>
```

## Perché questo esempio è così didattico

Perché mostra perfettamente il ruolo del converter:
- il modello resta pulito e usa un `bool`;
- la UI riceve il tipo che si aspetta (`Visibility`);
- la trasformazione resta nel livello di presentazione.

## Perché `Collapsed` è spesso preferito a `Hidden`

`Collapsed` nasconde l'elemento e rimuove anche lo spazio occupato nel layout.
`Hidden` lo nasconde ma mantiene lo spazio.

Questa scelta è importante anche dal punto di vista visivo.

## Quando usare questo approccio

Usalo quando:
- hai una condizione booleana;
- la UI deve mostrare o nascondere un elemento;
- non vuoi sporcare il modello con proprietà direttamente di tipo `Visibility`.

## Errori tipici

### Mettere `Visibility` nel modello quando basta un `bool`
Così il modello assorbe dettagli di presentazione non necessari.

### Dimenticare di registrare il converter come risorsa
Il binding non riuscirà a trovarlo.

### Non capire la differenza tra `Collapsed` e `Hidden`
La UI si comporta diversamente nel layout.

## Regola pratica

Il converter `bool` → `Visibility` è uno degli esempi migliori per capire i converter:
un dato semplice del modello viene adattato a un bisogno specifico della UI, senza confondere i ruoli.

# 03.03.07 - Esempio con `ListBox`

## Obiettivo dell'esempio

Qui vogliamo vedere un caso concreto e completo con:
- `ObservableCollection<T>`;
- elementi che implementano `INotifyPropertyChanged`;
- `ListBox` bindata alla collezione;
- aggiunta di elementi;
- modifica di un elemento già presente.

Questo esempio è molto utile perché mostra in modo visibile la differenza tra notifica della collezione e notifica della proprietà.

## Il modello dell'elemento

```csharp
using System.ComponentModel;
using System.Runtime.CompilerServices;

public class Cliente : INotifyPropertyChanged
{
    private string _nome = "";

    public string Nome
    {
        get => _nome;
        set
        {
            if (_nome != value)
            {
                _nome = value;
                OnPropertyChanged();
            }
        }
    }

    public event PropertyChangedEventHandler? PropertyChanged;

    protected void OnPropertyChanged([CallerMemberName] string? propertyName = null)
    {
        PropertyChanged?.Invoke(this, new PropertyChangedEventArgs(propertyName));
    }
}
```

## La finestra

```csharp
using System.Collections.ObjectModel;

public partial class MainWindow : Window
{
    public ObservableCollection<Cliente> Clienti { get; } = new();

    public MainWindow()
    {
        InitializeComponent();

        Clienti.Add(new Cliente { Nome = "Anna" });
        Clienti.Add(new Cliente { Nome = "Luca" });

        DataContext = this;
    }

    private void AggiungiCliente_Click(object sender, RoutedEventArgs e)
    {
        Clienti.Add(new Cliente { Nome = "Nuovo cliente" });
    }

    private void RinominaPrimo_Click(object sender, RoutedEventArgs e)
    {
        if (Clienti.Count > 0)
        {
            Clienti[0].Nome = "Cliente rinominato";
        }
    }
}
```

## Lo XAML

```xml
<StackPanel Margin="20">
    <ListBox ItemsSource="{Binding Clienti}"
             DisplayMemberPath="Nome"
             Height="180"
             Width="240"
             Margin="0,0,0,12"/>

    <StackPanel Orientation="Horizontal">
        <Button Content="Aggiungi"
                Width="100"
                Margin="0,0,8,0"
                Click="AggiungiCliente_Click"/>

        <Button Content="Rinomina primo"
                Width="120"
                Click="RinominaPrimo_Click"/>
    </StackPanel>
</StackPanel>
```

## Che cosa mostra davvero questo esempio

### 1. L'aggiunta alla collezione aggiorna la `ListBox`
Quando premi “Aggiungi”, compare una nuova riga.

Questo succede grazie a `ObservableCollection<T>`.

### 2. La modifica della proprietà aggiorna il testo mostrato
Quando premi “Rinomina primo”, il primo elemento cambia testo nella lista.

Questo succede grazie a `INotifyPropertyChanged` implementato su `Cliente`.

## Perché è un esempio importante

Perché fa vedere chiaramente due livelli:
- la lista reagisce ai cambiamenti della collezione;
- gli elementi reagiscono ai cambiamenti delle loro proprietà.

È uno dei migliori esempi per fissare il modello mentale corretto.

## Una variante ancora più istruttiva

Se togli `INotifyPropertyChanged` da `Cliente`, vedrai che:
- “Aggiungi” continua a funzionare;
- “Rinomina primo” non aggiorna la UI come previsto.

Questo test è molto formativo.

## Quando un esempio del genere è utile nel lavoro reale

È utile in:
- elenchi clienti;
- elenchi articoli;
- task list;
- liste di righe documento;
- pannelli con raccolte modificabili.

## Errori tipici

### Bindare la `ListBox` a una `List<T>` e aspettarsi aggiunte automatiche
Non è la scelta giusta per scenari dinamici.

### Usare `ObservableCollection<T>` ma non notificare le proprietà degli elementi
La lista si aggiorna solo a metà.

### Pensare che `DisplayMemberPath` cambi il modello di notifica
No: riguarda solo la presentazione.

## Regola pratica

Con una `ListBox`, se vuoi che:
- compaiano nuovi elementi;
- e si aggiornino anche i nomi degli elementi esistenti,

allora ti servono spesso insieme `ObservableCollection<T>` e `INotifyPropertyChanged`.

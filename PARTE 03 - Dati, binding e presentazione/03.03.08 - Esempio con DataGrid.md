# 03.03.08 - Esempio con `DataGrid`

## Perché il `DataGrid` è un test molto realistico

Se la `ListBox` è utile per capire il principio, il `DataGrid` è ancora più interessante perché rappresenta molto meglio scenari lavorativi reali.

In una griglia, di solito vuoi:
- vedere più colonne;
- aggiungere righe;
- modificare valori;
- aggiornare la UI in modo coerente.

Qui la combinazione tra `ObservableCollection<T>` e `INotifyPropertyChanged` mostra tutto il suo valore.

## Il modello

```csharp
using System.ComponentModel;
using System.Runtime.CompilerServices;

public class RigaOrdine : INotifyPropertyChanged
{
    private string _articolo = "";
    private int _quantità;
    private decimal _prezzo;

    public string Articolo
    {
        get => _articolo;
        set
        {
            if (_articolo != value)
            {
                _articolo = value;
                OnPropertyChanged();
            }
        }
    }

    public int Quantità
    {
        get => _quantità;
        set
        {
            if (_quantità != value)
            {
                _quantità = value;
                OnPropertyChanged();
            }
        }
    }

    public decimal Prezzo
    {
        get => _prezzo;
        set
        {
            if (_prezzo != value)
            {
                _prezzo = value;
                OnPropertyChanged();
            }
        }
    }

    public event PropertyChangedEventHandler? PropertyChanged;

    protected void OnPropertyChanged([CallerMemberName] string? propertyName = null)
    {
        PropertyChanged?.Invoke(this, new PropertyChangedEventArgs(propertyName));
    }
}
```

## La finestra

```csharp
using System.Collections.ObjectModel;

public partial class MainWindow : Window
{
    public ObservableCollection<RigaOrdine> Righe { get; } = new();

    public MainWindow()
    {
        InitializeComponent();

        Righe.Add(new RigaOrdine { Articolo = "Penna", Quantità = 2, Prezzo = 1.50m });
        Righe.Add(new RigaOrdine { Articolo = "Quaderno", Quantità = 5, Prezzo = 3.20m });

        DataContext = this;
    }

    private void AggiungiRiga_Click(object sender, RoutedEventArgs e)
    {
        Righe.Add(new RigaOrdine
        {
            Articolo = "Nuovo articolo",
            Quantità = 1,
            Prezzo = 0m
        });
    }

    private void ModificaPrimaRiga_Click(object sender, RoutedEventArgs e)
    {
        if (Righe.Count > 0)
        {
            Righe[0].Quantità += 1;
        }
    }
}
```

## Lo XAML

```xml
<StackPanel Margin="20">
    <DataGrid ItemsSource="{Binding Righe}"
              AutoGenerateColumns="False"
              CanUserAddRows="False"
              Height="220"
              Margin="0,0,0,12">
        <DataGrid.Columns>
            <DataGridTextColumn Header="Articolo" Binding="{Binding Articolo}" Width="*" />
            <DataGridTextColumn Header="Quantità" Binding="{Binding Quantità}" Width="100" />
            <DataGridTextColumn Header="Prezzo" Binding="{Binding Prezzo}" Width="100" />
        </DataGrid.Columns>
    </DataGrid>

    <StackPanel Orientation="Horizontal">
        <Button Content="Aggiungi riga"
                Width="110"
                Margin="0,0,8,0"
                Click="AggiungiRiga_Click"/>

        <Button Content="Incrementa quantità"
                Width="140"
                Click="ModificaPrimaRiga_Click"/>
    </StackPanel>
</StackPanel>
```

## Che cosa dimostra questo esempio

### Aggiunta di righe
Quando aggiungi una riga, il `DataGrid` la mostra subito.
Questo dipende da `ObservableCollection<T>`.

### Modifica dei dati interni
Quando incrementi la quantità della prima riga, la cella si aggiorna.
Questo dipende da `INotifyPropertyChanged` sull'elemento `RigaOrdine`.

## Perché questo esempio è molto importante

Perché rappresenta uno scenario molto vicino al lavoro reale:
- ordini;
- preventivi;
- documenti;
- righe di magazzino;
- liste operative.

Qui non stai più guardando solo teoria di binding.
Stai guardando il comportamento di una schermata gestionale concreta.

## Un punto da osservare bene

Se togli `INotifyPropertyChanged` dal tipo `RigaOrdine`, vedrai che:
- le righe nuove compariranno comunque;
- ma le modifiche interne alle celle non avranno lo stesso comportamento dinamico.

Questa è una delle prove pratiche più utili da fare.

## Errori tipici

### Pensare che il `DataGrid` “faccia tutto da solo”
No: la qualità dell'aggiornamento dipende sempre dal modello dati.

### Usare una lista non osservabile
Le nuove righe non si riflettono bene nella griglia.

### Non notificare le proprietà delle righe
Le celle non reagiscono correttamente ai cambiamenti.

## Regola pratica

Se una griglia deve vivere davvero, devi pensare a due livelli:
- la collezione delle righe;
- gli oggetti che rappresentano ogni riga.

Entrambi devono notificare nel modo giusto.
